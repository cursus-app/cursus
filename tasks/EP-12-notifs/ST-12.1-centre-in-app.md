## Contexte business

Centre de notifications in-app dans le header. Sans cloche centralisée, l'utilisateur rate les notifs importantes (alertes, validations, etc.).

Valeur livrée : cloche dans le header avec compteur, panneau déroulant avec liste, actions "Marquer lues" et "Supprimer", subscription Realtime.

## Description fonctionnelle

Cloche dans le header avec compteur de notifs non lues. Panneau déroulant avec la liste : type, titre, date, action de redirection. "Marquer lues" + "Supprimer". Subscription Realtime pour push.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Réception en temps réel**

```gherkin
Given un utilisateur reçoit une notif
Then une cloche dans le header affiche un compteur incrémenté
And cliquer sur la cloche ouvre un panneau avec la liste
And chaque notif a : type, titre, date, action de redirection
```

**Scénario 2 — Marquer comme lue**

```gherkin
Given l'utilisateur ouvre le panneau
When il clique sur une notif
Then elle est marquée lue
And le compteur décrémente
```

**Scénario 3 — Actions globales**

```gherkin
Given il a 5 notifs non lues
When il clique "Marquer toutes lues"
Then toutes passent en lues
And le compteur passe à 0
```

## Cas limites à traiter

- **0 notif** : empty state empathique.
- **100+ notifs** : pagination.
- **Compteur > 99** : afficher "99+".
