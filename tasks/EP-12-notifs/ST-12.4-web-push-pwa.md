## Contexte business

Web Push pour les notifs urgentes (PWA prep). Feature **premium**. Prépare le terrain pour la PWA complète (EP-24).

Valeur livrée : opt-in push notifications natives navigateur, respect des quiet hours et DND, clic ouvre l'app sur le bon écran.

## Description fonctionnelle

Service worker + subscription Web Push (VAPID keys). Endpoint d'envoi. Préférences (quiet hours). Respect prefers-reduced-motion + DND système.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Opt-in et push**

```gherkin
Given un user accepte les push notifications
When un événement urgent survient
Then un push web est envoyé via Web Push API
And clic sur la notif ouvre l'app sur le bon écran
```

**Scénario 2 — Respect DND / reduced motion**

```gherkin
Given prefers-reduced-motion ou DND système actif
Then respecter (pas de notif intrusive)
```

**Scénario 3 — Quiet hours**

```gherkin
Given user a configuré quiet hours 22h-8h
When une notif arrive à 23h
Then elle est mise en attente jusqu'à 8h
```

## Cas limites à traiter

- **Browser sans support Push** : graceful degradation.
- **User désinstalle l'app** : subscription expirée → cleanup.
- **Service worker non actif** : pas de push.
