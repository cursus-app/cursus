## Contexte business

L'admin (Mohamed, puis sa hiérarchie) a besoin de **rendre compte** de l'activité Cursus à des décideurs qui ne se connecteront jamais à l'app : direction RH, responsable formation, comité pédagogique. Ces stakeholders veulent une feuille Excel par trimestre — pas un dashboard. Cette story livre l'exhaust : un export CSV agrégé qui montre la valeur du produit en chiffres (cohortes, complétions, certificats).

L'export est aussi utilisé en interne pour des analyses ad-hoc et pour la conformité (audit de présence de données avant suppression RGPD).

## Description fonctionnelle

Page "Administration > Reporting > Export". L'admin sélectionne un **type d'export** (Cohortes, Stagiaires, Soumissions, Certificats), une **période** (date range picker), et des **filtres optionnels** (cursus, statut). Au clic sur "Générer", un job Inngest est enqueué qui produit un fichier CSV (UTF-8 BOM pour Excel, séparateur `,`) stocké dans Supabase Storage avec une URL signée 7 jours, puis un email Resend envoie le lien à l'admin.

L'écran affiche l'historique des 10 derniers exports avec statut (`running`, `ready`, `expired`), taille, bouton "Re-générer".

## Critères d'acceptation (Gherkin)

**Scénario 1 — Génération d'un export cohortes**

```gherkin
Given un admin connecté sur la page "Reporting > Export"
And il sélectionne le type "Cohortes" et la période "Q1 2026"
When il clique sur "Générer l'export"
Then un job Inngest "report.generate" est enqueué avec l'ID admin et les filtres
And l'UI affiche une ligne d'export en statut "En cours" avec un spinner
And dans les 60 secondes le statut passe à "Prêt"
And un email Resend est envoyé à l'admin avec un lien signé
And le CSV contient les colonnes : cohorte_id, titre, cursus, date_debut, date_fin, nb_stagiaires, taux_completion, taux_validation_capstone
```

**Scénario 2 — Export expiré**

```gherkin
Given un export généré il y a 8 jours
When l'admin clique sur le lien dans l'historique
Then la page affiche "Cet export a expiré. Re-génère-le pour obtenir un nouveau lien."
And le bouton "Re-générer" relance un job avec les mêmes paramètres
```

**Scénario 3 — Volume excessif**

```gherkin
Given un admin tente d'exporter "Soumissions" sur 24 mois
And le volume estimé dépasse 100 000 lignes
When il clique sur "Générer"
Then un message d'avertissement s'affiche "Volume estimé : 120k lignes. Réduis la période ou utilise un filtre."
And l'export n'est pas lancé tant que l'admin ne réduit pas le scope
```

## Cas limites à traiter

- **Aucune donnée sur la période** : CSV avec uniquement l'en-tête + email mentionnant "Aucune donnée trouvée".
- **Crash du job** : Inngest retry 2× ; au-delà, statut "Erreur" avec bouton "Re-tenter" + log Sentry.
- **Admin déconnecté pendant la génération** : le job continue ; l'email arrive de toute façon.
- **Caractères spéciaux (virgules, guillemets, retours ligne)** : escaping CSV correct (RFC 4180).
- **Données supprimées RGPD entre temps** : le CSV reflète l'état au moment de la génération ; les comptes supprimés apparaissent comme `user_id` UUID + `(supprimé)`.
- **Fichier > 10 Mo** : compression gzip et CSV stocké comme `.csv.gz`.

## Observabilité

- **Logs** :

  - `report.generate.started` (info : `job_id`, `type`, `period`)
  - `report.generate.completed` (info : `job_id`, `rows_count`, `bytes`, `duration_ms`)
  - `report.generate.failed` (error : `job_id`, `error`)

- **Métriques** :

  - Histogram `cursus.reports.generation.duration.seconds` labellisé par `type`
  - Compteur `cursus.reports.generated.total` / `.failed.total`

- **Alertes** : taux d'échec > 5 % sur 1h.

## Considérations sécurité

- RBAC `admin` strict sur toutes les routes.
- URLs Supabase Storage signées HMAC (expiration 7 jours).
- Pas de PII en clair dans les logs (UUIDs uniquement).
- Path traversal impossible : bucket isolé par `<admin_id>/`.
- Audit log : toute génération est tracée (`report.generated`).

## Considérations accessibilité

- Formulaire : labels explicites, helper text, erreurs annoncées.
- Tableau historique : `<caption>` "Historique de tes exports", colonnes triables.
- Téléchargement : annonce `aria-live` "Téléchargement de l'export en cours".

## Considérations performance

- Streaming CSV via cursor Prisma `findMany({ cursor, take })` (pas de chargement total en mémoire).
- Index DB via `@@index([startDate, endDate])` sur `Cohorte`, `@@index([createdAt])` sur `Submission`.
- Jobs Inngest plafonnés à 5 concurrents.
- Cible : export 10k lignes en < 30 s.
