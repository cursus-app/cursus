## Contexte business

Un co-formateur peut être assigné à une cohorte entière ou uniquement à certains modules (ex : Aïcha sur le module cybersécurité). Sans cette finesse, soit on partage tout (perte de focus pour le co-formateur), soit on crée des sous-cohortes (complexité générale).

Valeur livrée : assignation ciblée qui respecte l'expertise des co-formateurs, isolation des alertes par module.

## Description fonctionnelle

Section "Équipe" dans la fiche cohorte. Ajout d'un co-formateur par email ou recherche. Choix : global (toute la cohorte) ou limité à N modules. Notification au co-formateur. Adaptation RLS pour limiter par modules.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Ajout d'un co-formateur**

```gherkin
Given un formateur principal est sur une cohorte
When il ajoute Aïcha comme co-formateur
Then Aïcha reçoit une notif (in-app + email)
And elle peut accéder à la cohorte avec rôle co_formateur
```

**Scénario 2 — Limitation par module**

```gherkin
Given le formateur principal limite Aïcha au module "Cybersec basics"
When Aïcha consulte la cohorte
Then elle voit uniquement les livrables de ce module dans ses alertes
And RLS bloque l'accès aux autres modules
```

**Scénario 3 — Retrait**

```gherkin
Given un co-formateur est ajouté
When le formateur principal le retire
Then son accès est révoqué immédiatement
And il est redirigé hors de la cohorte avec message
```

## Cas limites à traiter

- **Co-formateur déjà présent** : message d'info, pas d'erreur.
- **Co-formateur enlevé pendant qu'il consulte** : redirection avec message.
- **Multi-cohortes** : co-formateur peut être dans plusieurs cohortes.
- **Modification des modules assignés** : RLS rechargeée à chaque requête.

## Non-goals (hors scope)

- Pas de notifications fines par co-formateur au MVP (héritent des préférences globales).
- Pas de hiérarchie co-formateur senior/junior.
- Pas de délégation temporaire (vacances).

## Tests à écrire

- **Tests RLS** : co-formateur module M1 ne voit pas livrables module M2.
- **E2E (Playwright)** : ajout/retrait co-formateur.
- **Integration** : co-formateur multi-cohortes voit bien chaque scope.
- **Security** : co-formateur ne peut pas s'auto-promouvoir formateur principal.

## Observabilité

- **Logs** : `cohorte.co_formateur.added`, `cohorte.co_formateur.removed`.
- **Métriques** : nombre moyen de co-formateurs par cohorte.

## Considérations sécurité

- Audit log sur add/remove.
- Co-formateur ne peut pas inviter d'autres co-formateurs.

- Liste co-formateurs : focus ring visible.

## Considérations performance

- RLS check rapide grâce à index sur (cohorte_id, user_id, module_id).
