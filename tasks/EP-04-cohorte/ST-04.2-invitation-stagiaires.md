## Contexte business

Le formateur invite des stagiaires un par un (cas standard) ou en masse via CSV (rentrée scolaire avec 30 élèves). Sans bulk import, le formateur passe 30 min à cliquer pour onboarder une promo.

Valeur livrée : invitation en masse en 1 min, dans la même UX que single (modal unique avec deux modes).

## Description fonctionnelle

Modal "Inviter des stagiaires" avec champ multi-emails (coller virgule-séparé) + upload CSV (PapaParse, colonnes email/prenom/nom). Réutilise le service d'invitation ST-02.2. Liste des invitations envoyées avec statut. Bouton "Renvoyer" si pas de réponse en 3j.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Multi-emails collés**

```gherkin
Given un formateur ouvre la modal d'invitation
When il colle "karim@ex.com, sarah@ex.com, ahmed@ex.com"
Then 3 invitations sont créées et envoyées
And la liste s'affiche avec statut "Invité" pour chacun
```

**Scénario 2 — Upload CSV**

```gherkin
Given il upload un CSV avec colonnes email,prenom,nom
When le format est valide
Then les invitations sont créées avec les noms pré-remplis
And un résumé "30 invitations envoyées, 2 doublons ignorés" s'affiche
```

**Scénario 3 — Renvoi**

```gherkin
Given une invitation reste sans réponse depuis 3 jours
When le formateur consulte la liste
Then un bouton "Renvoyer" est visible
And cliquer renvoie l'email
```

## Cas limites à traiter

- **Emails dupliqués dans la même requête** : déduplication automatique.
- **Email déjà membre d'une autre cohorte** : autorisé (multi-cohortes).
- **CSV malformé** : message d'erreur précis (ligne, colonne).
- **Bulk > 50 emails** : confirmation requise.
- **Email déjà membre de cette cohorte** : ignoré avec mention.
- **Email invalide** : ignoré avec mention.

## Observabilité

- **Logs** : `invitation.batch.sent`, `invitation.batch.deduplicated`.
- **Métriques** : `invitations.batch.total`, taille moyenne des batchs (cible 15-25).

## Considérations sécurité

- Rate limit 50 invitations/h/formateur (anti-spam).
- CSV : pas d'échappement d'injection CSV (Excel formula).
- Validation Zod sur tous les emails.

## Considérations accessibilité

- Drag-drop CSV avec alternative bouton.
- Résumé d'invitation : `aria-live="polite"`.
- Erreurs CSV affichées ligne par ligne avec contexte.

## Considérations performance

- Batch 30 invitations : < 5s (async via Inngest si > 10).
- Parser CSV : streaming pour > 100 lignes.

## Definition of Done spécifique

- E2E batch 30 stagiaires (CSV).
- Rate limit testé : 51ᵉ invitation = 429.
- CSV avec emails accentés correctement géré.
