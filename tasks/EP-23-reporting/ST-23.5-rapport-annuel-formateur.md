## Contexte business

Un formateur expérimenté qui a livré 10+ cohortes sur 2 ans veut une vue annuelle synthétique : nb stagiaires formés, cohortes lancées, taux de réussite, évolution. Outil de marketing personnel (publié sur LinkedIn) et de bilan interne.

Business value : reconnaissance professionnelle + rétention formateur ("Cursus me donne mon rapport annuel auto"). Effet réseau / acquisition.

## Description fonctionnelle

- Page `/formateur/rapport-annuel?year=2025`.
- Sections :

  - **Hero** : "En 2025, tu as formé X stagiaires sur Y cohortes".
  - **Stats** : KPI principaux (heures de formation cumulées, taux de réussite moyen, médailles attribuées).
  - **Top moments** : cohortes les plus performantes, stagiaires distingués (avec accord).
  - **Évolution** : line chart de progression vs année N-1.
  - **Insights** : modules les plus appréciés (via quiz NPS si dispo), tendances.

- Génération PDF (réutilise ST-23.4).
- Partage social (image OG card).

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Rapport année active
Given Mohamed a livré 4 cohortes en 2025
When il ouvre /formateur/rapport-annuel?year=2025
Then la page affiche Hero, Stats, Top moments, Évolution, Insights
And les stats sont calculées à partir des cohortes auxquelles il a participé
And un CTA "Exporter en PDF" est présent
```

```gherkin
Scénario 2 — Année sans cohorte
Given Mohamed n'a pas eu de cohorte en 2024
When il ouvre /formateur/rapport-annuel?year=2024
Then un message empathique "Pas de cohorte cette année" s'affiche
And un CTA "Voir 2025" est proposé
```

```gherkin
Scénario 3 — Comparaison N-1
Given 2024 et 2025 ont des données
When le rapport 2025 est généré
Then la section Évolution montre delta vs 2024 (ex: +30 % stagiaires)
And badge ↑ / ↓ apparent
```

## Cas limites à traiter

- **Année future** : 404 / message "Année non disponible".
- **Année partielle (en cours)** : disclaimer "Année en cours, résultats provisoires".
- **Stagiaires opt-out du nom public** : afficher "Un stagiaire" anonymisé.
- **Premier rapport** (pas de N-1) : section évolution cachée.
- **Très haute volumétrie (100+ cohortes)** : agrégations efficaces, pas de scan brut.

## Dépendances

- **Bloqué par** : ST-23.1 (graphes), ST-23.4 (PDF), ST-08.\* (stats).
- **Bloque** : aucune.

## Non-goals

- Pas de rapport mensuel / trimestriel (juste annuel).
- Pas de personnalisation visuelle (template fixe).
- Pas de gamification dans le rapport (juste stats).
- Pas de comparaison cross-formateurs.

## Tests à écrire

- **Unit Vitest** : agrégations annuelles correctes sur fixture.
- **Integration** : endpoint retourne payload structuré.
- **A11y axe-core** : sections semantically structured.
- **E2E Playwright** : naviguer, switcher year, exporter PDF.
- **i18n** : FR + EN.
- **Visual Chromatic** : hero + sections.
- **Security** : RLS, autre formateur ne voit pas les stats.

## Observabilité

- **Logs** : event `rapport_annuel.viewed` (info) avec `{ user_id, year, exported_pdf }`.
- **Métriques** : compteur views par year, taux d'export PDF.
- **Insight** : taux de partage social.

## Considérations sécurité

- RLS strict.
- Pas de leak de noms stagiaires non opt-in.
- OG card image ne contient pas PII sensible.

## Considérations accessibilité

- Hero : titre `<h1>`, animation respectant prefers-reduced-motion.
- Sections : `<section aria-labelledby>`.
- Charts avec fallback table.
- CTA navigable au clavier.

## Considérations performance

- Vue agrégée pré-calculée.
- Cache 1 h (les stats annuelles changent peu).
- Cible : page interactive < 800 ms.
- PDF généré en < 20 s.
