## Contexte business

Une heatmap "jour x heure" révèle les patterns d'engagement de la cohorte : pics d'activité (lundi matin = méthodologique ?), creux (vendredi après-midi = décrochage ?). Le formateur peut adapter sa présence (faire les office hours quand l'activité monte). Outil de pilotage qualitatif puissant.

Business value : Differentiator de pilotage premium. Aucune LMS classique ne propose une heatmap d'engagement par cohorte.

## Description fonctionnelle

- Composant `<ActivityHeatmap />` : 7 lignes (jours de la semaine) × 24 colonnes (heures). Chaque cellule colorée selon intensité (nb d'événements : soumissions, login, quiz attempts).
- Page `/cohortes/:id/heatmap` ou intégré dans `/cohortes/:id/reporting`.
- Filtres : date range (30 / 90 / 180 derniers jours), type d'événement (soumissions / login / quiz).
- Couleur : palette séquentielle (par ex viridis), respectueuse accessibilité.
- Tooltip au hover : "Mercredi 14h : 23 événements".
- Export PNG (via html2canvas) ou JSON brut.

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Heatmap soumissions
Given une cohorte avec 60 jours d'activité
When le formateur ouvre la heatmap (events: soumissions, range: 60 j)
Then la heatmap affiche 168 cellules colorées selon l'intensité
And hover sur une cellule affiche "Mercredi 14h : 23 soumissions"
And un légende color-scale est présente
```

```gherkin
Scénario 2 — Filtre type d'event
Given la heatmap est active
When je switch "soumissions" → "login"
Then la heatmap se recalcule et change de pattern
And un légende update son label
```

```gherkin
Scénario 3 — Pas d'activité
Given une cohorte créée hier (pas encore d'événements)
When je consulte la heatmap
Then une empty state apparaît "Pas encore de données"
```

## Cas limites à traiter

- **Timezone** : utiliser la timezone du formateur (DST aware).
- **Heures creuses (2h-6h)** : afficher quand même, cellules très claires.
- **Pic anormal** (1 cellule avec 100× l'intensité moyenne) : log scale optionnelle.
- **Petite cohorte avec peu de données** : agrégation plus large (groupes 4h).
- **Couleurs daltoniens** : utiliser viridis (color-blind safe).
- **Mobile** : heatmap scrollable horizontalement (ou vertical fix).

## Considérations sécurité

- RLS.
- Pas de leak cross-cohort.

## Considérations accessibilité

- SVG heatmap avec `<table>` équivalente cachée visuellement (mais lue par screen reader).
- Chaque cellule a un `aria-label` "Mercredi 14h, 23 événements".
- Navigation flèches dans la grille.
- Palette accessible (viridis).
- Légende color-scale explicite + symboles.

## Considérations performance

- Vue matérialisée nuit (`mv_cohort_heatmap`).
- Pas de fetch en streaming (gros payload évité, 168 floats = 1 KB).
- SVG rendu chaîne (pas par cell pour éviter 168 mounts).
- Cible : interactive < 500 ms post fetch.
