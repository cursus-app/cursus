## Contexte business

Un formateur ou un admin qui pilote plusieurs cohortes consécutives veut comparer : la cohorte 5 est-elle plus performante que la 4 ? Pourquoi ? Cette story livre la vue comparative cross-cohortes : taux de complétion, mention, dropouts, temps moyen, etc.

Business value : Differentiator de pilotage stratégique. Un formateur expert peut itérer sur son cursus et mesurer l'effet (data-driven).

## Description fonctionnelle

- Page `/admin/comparison` (admin) ou `/formateur/comparison` (formateur).
- Sélecteur multi : 2-5 cohortes à comparer.
- KPI side-by-side : taux complétion, taux mention bien+, taux dropout, temps moyen jusqu'à capstone, nb soumissions / stagiaire.
- Graphe 1 : bar chart KPI side-by-side.
- Graphe 2 : line chart "progression cohortes alignées sur jour 0".
- Tableau détaillé exportable CSV.
- Highlight des écarts significatifs (delta > 15 %).

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Comparer 3 cohortes
Given 3 cohortes terminées
When le formateur les sélectionne dans le comparateur
Then le bar chart affiche les 5 KPI côte à côte
And les écarts > 15 % sont mis en évidence (badge ↑ ou ↓)
And un line chart aligne les 3 trajectoires sur "jour 0"
```

```gherkin
Scénario 2 — Cohortes en cours
Given 1 cohorte terminée + 1 en cours
When je les compare
Then les KPI de la cohorte en cours sont marqués "partiels"
And un disclaimer apparaît "Cohorte en cours, résultats provisoires"
```

```gherkin
Scénario 3 — Export CSV
Given 3 cohortes comparées
When je clique "Exporter CSV"
Then un fichier avec colonnes : kpi, cohorte_1, cohorte_2, cohorte_3, delta_2_1, delta_3_1
And encoding UTF-8 BOM
```

## Cas limites à traiter

- **Cohortes incomparables** (durées différentes) : normalisation par jour de progression (% de la durée totale).
- **Une seule cohorte sélectionnée** : pas de comparaison, message "Sélectionnez au moins 2 cohortes".
- **Trop de cohortes** (> 5) : limite ferme à 5, sinon graphe illisible.
- **Cohortes de cursus différents** : avertissement "Comparaison cross-cursus".
- **Données manquantes** : afficher "—" et tooltip "Données indisponibles".
- **RLS** : un formateur ne voit que SES cohortes ; un admin voit toutes.

## Non-goals

- Pas d'analyse causale ("pourquoi cohorte X est meilleure") au MVP.
- Pas d'A/B test entre cohortes.
- Pas de filtrage par module pour compare au MVP.
- Pas de comparaison cross-orgs.

## Tests à écrire

- **Unit Vitest** : `highlightDelta` retourne signe correct.
- **Integration** : endpoint avec 3 cohort_ids retourne payload structuré.
- **RBAC** : formateur reçoit seulement ses cohortes éligibles.
- **A11y axe-core** : compareur navigable au clavier.
- **E2E Playwright** : sélectionner 3 cohortes, voir comparison, exporter.
- **Visual Chromatic** : snapshots.

## Observabilité

- **Logs** : event `reporting.compare.run` (info) avec `{ cohort_ids, user_role }`.
- **Métriques** : compteur compares, top KPIs les plus comparés.
- **Insight** : taux d'usage par formateur (engagement avec la feature).

## Considérations sécurité

- RLS strict.
- Pas de leak d'autres orgs.
- CSV export rate-limité.

## Considérations accessibilité

- Tableau comparatif HTML accessible (table sémantique).
- Charts avec fallback table.
- Highlight delta : badge + symbole + texte (pas couleur seule).
- Lecteur d'écran annonce le contexte des deltas.

## Considérations performance

- Vue Postgres pré-calculée.
- Cache 10 min sur les KPIs.
- Lazy-load charts.
- Cible : interactive < 1 s.
