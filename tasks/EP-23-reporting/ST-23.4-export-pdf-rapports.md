## Contexte business

Le CSV (ST-14.2) couvre les besoins comptables / data analysts, mais pour partager avec un client / un comité / une institution, un PDF mis en page est attendu : couverture, KPI clés, graphes inline, conclusion. Cette story livre l'export PDF des rapports de reporting (cohorte ou compare).

Business value : Differentiator B2B (partage simple par email avec PDF attaché, format universel).

## Description fonctionnelle

- Bouton "Exporter PDF" sur les pages reporting (cohorte single, compare multi).
- Backend : job Inngest `pdf.report.generate` (async, peut prendre 10-30 s).
- Template PDF : couverture (titre, dates, logo Cursus + logo org si dispo), section KPI, graphes (SVG vectoriel), table détaillée, footer (page X/Y, confidentiality notice).
- Stockage temporaire (Supabase Storage 24 h) avec lien signé.
- Notification "PDF prêt" via toast + email.
- Locale FR/EN selon utilisateur.

## Cas limites à traiter

- **Très grosse cohorte (200 stagiaires)** : PDF tronqué à 50 lignes + annexe "voir CSV".
- **Cursus encore vide** : message PDF "Pas de données".
- **Lien expiré** : 410 propre + invitation à régénérer.
- **Génération échoue** : retry 1 fois, notification erreur si échec persistant.
- **Bandwidth limité** : PDF compressé (cible < 2 MB).
- **Branding personnalisé** (logo org) : prévu en ST-14.3.

- **TT-23.4.7** — Endpoint `GET /api/exports/:id` pour récupérer statut + lien.
- **TT-23.4.8** — Toast + email notification "PDF prêt".

## Non-goals

- Pas de PDF interactif (juste statique).
- Pas de signature numérique du PDF au MVP.
- Pas de génération à la volée (toujours async).
- Pas de support customisé par client (juste branding logo).

## Tests à écrire

- **Unit Vitest** : template render correct (snapshot HTML / structure).
- **Integration** : job généré → buffer PDF non vide → upload Supabase.
- **E2E Playwright** : formateur clique → attendu toast → DL → vérifier page count + titre.
- **Security** : RLS sur génération + lien signé strict.
- **Performance** : génération < 30 s pour cohorte 50 stagiaires.
- **i18n** : FR + EN snapshots.
- **A11y du PDF** : tagged PDF (PDF/UA), structure correcte.

## Observabilité

- **Logs** : `pdf.report.generated` (info) avec `{ cohort_id, latency_ms, size_bytes, user_id }`.
- **Métriques** : compteur generations, p95 latency.
- **Alerte** : taux d'échec > 5 %.

## Considérations sécurité

- Lien signé 24 h max.
- RLS strict avant génération.
- Pas de PII sensible (numéro stagiaire interne OK, pas email).
- Audit log de la génération.
- Rate limit (max 5 exports / heure / user).

## Considérations accessibilité

- PDF tagué (PDF/UA-1) : structure heading, alt text sur charts.
- Lecteurs d'écran peuvent parcourir le PDF.
- Contraste typo respecté.

## Considérations performance

- Job async (pas blocage HTTP).
- Cible : génération < 30 s pour 50 stagiaires.
- PDF compressé (cible < 2 MB).
- Stockage temporaire 24 h, purge auto.
- SVG vectoriels pour graphes (pas PNG).
