## Contexte business

Le formateur veut voir et résoudre les alertes en 1-2 clics. Sans UI dédiée, les alertes restent invisibles ou en email et l'efficacité du suivi asynchrone s'écroule.

Valeur livrée : page `/alertes` filtrable, contexte complet par alerte, 5 actions principales accessibles (commenter, override, étendre, planifier, marquer traitée).

## Description fonctionnelle

Page `/alertes` avec liste filtrable. Pour chaque alerte : contexte (livrable, dernier rapport harnais, message stagiaire) + actions (commenter, override harnais, étendre l'échéance, planifier un appel, marquer traitée).

## Critères d'acceptation (Gherkin)

**Scénario 1 — Liste avec filtres**

```gherkin
Given un formateur ouvre /alertes
When les alertes sont chargées
Then la liste affiche : stagiaire, type, ancienneté, statut, action principale
And des filtres : type, stagiaire, statut (ouverte/résolue)
```

**Scénario 2 — Fiche détaillée**

```gherkin
Given il clique sur une alerte
When la fiche s'ouvre
Then il voit le contexte (livrable, dernier rapport harnais, message stagiaire)
And des actions sont accessibles : Commenter, Override harnais, Étendre l'échéance, Planifier un appel, Marquer traitée
```

**Scénario 3 — Résolution**

```gherkin
Given l'alerte est marquée traitée
Then elle disparaît de la vue par défaut (filtre "ouvertes")
And entre dans l'historique
```

## Cas limites à traiter

- **Alerte créée auto puis stagiaire se débloque** : auto-résolution avec mention.
- **Alerte avec stagiaire désactivé** : reste visible mais en historique.
- **Co-formateur module-scoped** : voit uniquement alertes de ses modules.
