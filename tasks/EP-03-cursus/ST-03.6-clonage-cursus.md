## Contexte business

Le formateur veut créer une variante d'un cursus existant (ex : passer de "Cybersec L1" à "Cybersec L2") sans tout recréer. Sans clonage, c'est 2h de copier-coller manuel.

Valeur livrée : duplication 1-clic d'un cursus complet (modules, ressources, livrables, quiz) en brouillon.

## Description fonctionnelle

Bouton "Dupliquer" sur la fiche d'un cursus qui crée un brouillon avec toutes les données copiées. Le clone est indépendant : modifs n'impactent pas l'original (deep clone).

## Critères d'acceptation (Gherkin)

**Scénario 1 — Duplication complète**

```gherkin
Given un formateur est sur la fiche d'un cursus
When il clique "Dupliquer"
Then un nouveau cursus brouillon est créé avec suffixe " (copie)"
And tous les modules, ressources, livrables, quizs sont copiés
And il est redirigé sur l'édition du clone
```

**Scénario 2 — Indépendance des données**

```gherkin
Given un clone est créé à partir d'un cursus source
When on modifie un module du clone
Then le cursus source n'est PAS impacté
```

**Scénario 3 — Permissions du clone**

```gherkin
Given un formateur duplique un cursus dont il est l'auteur
Then il est auteur du clone
And seul lui peut l'éditer (RLS)
```

## Cas limites à traiter

- **Cursus en brouillon dupliqué** : autorisé (clone est brouillon aussi).
- **Cursus archivé dupliqué** : autorisé (clone réanime en brouillon).
- **Cursus avec ressources broken** : copiées telles quelles (broken aussi).
- **Nom > limite** après ajout " (copie)" : tronqué.

## Non-goals (hors scope)

- Pas de lien parent-enfant entre original et clone.
- Pas d'héritage des updates de l'original.
- Pas de duplication partielle (ex : juste les modules 1-5).

## Tests à écrire

- **E2E (Playwright)** : duplication produit bien tout le contenu.
- **Integration** : aucun pointer vers les données du cursus source (deep clone).
- **Security** : un formateur ne peut pas dupliquer un cursus d'un autre formateur sans permission.

## Observabilité

- **Logs** : `cursus.duplicated` avec source_id et clone_id.
- **Métriques** : `cursus.duplicated.total`.

## Considérations sécurité

- RLS : duplication requiert lecture du source + création autorisée.
- Pas de copie d'audit_log ni de submissions.

## Considérations accessibilité

- Bouton "Dupliquer" : label clair, confirmation accessible.

## Considérations performance

- Duplication async via job Inngest si > 50 modules.
- Sinon synchrone < 2s.

## Definition of Done spécifique

- E2E : duplication de cursus avec 10 modules + 50 ressources.
- Test indépendance : modifier clone n'impacte pas source.
