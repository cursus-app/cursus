## Contexte business

Le pilote n'a de valeur que si on en tire des apprentissages structurés. Cette story livre un rapport de pilote avec métriques quanti, feedback quali, recommandations. C'est l'input direct du Go/No-Go v1.0 (ST-17.6).

## Description fonctionnelle

Après fin de cohorte pilote : rapport `docs/pilot/postmortem.md` avec :

- **Métriques quanti** : taux d'engagement, taux validation 1ᵉ essai (cible : 40-60 %), taux capstone validé (cible : ≥ 70 %), temps formateur économisé (auto-déclaratif), erreurs Sentry par 1k sessions.
- **Feedback quali** : entretiens 30 min par stagiaire + Mohamed, questions standardisées.
- **Synthèse NPS** + verbatims significatifs.
- **Top 5 échec / friction**, **Top 5 succès**.
- **Rétroéquipe interne** : ce qui a bien marché, ce qui n'a pas.
- **Recommandations v1.0** (priorité P0/P1/P2).

## Critères d'acceptation (Gherkin)

**Scénario 1 — Métriques collectées**

```gherkin
Given le pilote est terminé
When on extrait les métriques produit
Then on a : taux validation 1ᵉ essai, taux capstone, DAU moyen, alertes/cohorte
And elles sont comparées aux cibles
```

**Scénario 2 — Entretiens stagiaires**

```gherkin
Given 5 stagiaires ont terminé
When on les interview 30 min chacun
Then feedback est capturé dans `docs/pilot/interviews/<stagiaire>.md`
And des verbatims significatifs sont extraits
```

**Scénario 3 — Rapport final**

```gherkin
Given toutes les données sont collectées
When Mohamed rédige le rapport
Then il contient : intro, méthode, métriques, feedback, top friction, top succès, recommandations
And il est partagé équipe + stakeholders
```

## Cas limites à traiter

- **Stagiaire indisponible pour entretien** : substitut par questionnaire écrit.
- **Données manquantes** (analytics down un jour) : noter, ne pas extrapoler.
- **Feedback contradictoires** : présenter les deux côtés.
- **Capstone non validé** : sujet sensible, traiter avec respect.
- **Drop-off** : interviewer les drop-out (impact significatif).
