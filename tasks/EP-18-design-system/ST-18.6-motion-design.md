## Contexte business

Le motion design est l'un des marqueurs Premium (axe 1). Bien exécuté = signal de qualité perceptible immédiatement. Mal exécuté = annoy + accessibility issue. Cette story implemente les patterns motion définis dans design-system §4.

## Description fonctionnelle

Micro-interactions signées :

- Transitions de page (fade subtil)
- Modals (slide depuis bas mobile / scale center desktop)
- Popovers (scale 0.95 → 1 + fade)
- Skeleton shimmer pour loading
- Bounce sur validation livrable + badge
- Stagger sur listes (10-30ms entre items)
- Toggle dark/light : crossfade 250ms
- Cmd+K palette : slide-down + fade

Lib : `motion-v` ou `@vueuse/motion`. Toutes les animations > 200ms wrappent dans `@media (prefers-reduced-motion: no-preference)`.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Validation livrable**

```gherkin
Given un harnais valide une soumission
When le badge XP apparaît
Then il a une animation bounce courte (300ms)
And un sparkle subtil l'accompagne
And les utilisateurs prefers-reduced-motion voient une fade simple
```

**Scénario 2 — Modal mobile**

```gherkin
Given la page est mobile
When une modal s'ouvre
Then elle slide depuis le bas en 300ms ease-out
And l'animation respecte prefers-reduced-motion
```

**Scénario 3 — Stagger liste cohorte**

```gherkin
Given la liste de stagiaires charge
When les rows apparaissent
Then elles staggerent 20ms entre chaque (max 10 visibles)
And au-dela les rows apparaissent direct
```

## Cas limites à traiter

- **prefers-reduced-motion** : toutes animations > 200ms désactivées.
- **Performance device faible** : pas d'animation coûteuse (shadows animées).
- **Long lists** : limit stagger à 10 premiers visibles.
- **Will-change overuse** : appliquer pendant l'animation seulement.
- **Z-index conflicts** : utiliser tokens z-index.

## Considérations accessibilité

- **CRITIQUE** : respecter `prefers-reduced-motion`.
- Pas de vestibular triggers (excess parallax, spinning).
- Animations < 200ms toujours visibles (acceptable).
- Annoncer dynamic content via `aria-live` indépendamment de l'anim.

## Considérations performance

- Animations GPU (transform, opacity), pas left/top/width.
- `will-change` activé pendant anim, retiré après.
- INP < 200ms (interaction starts anim).
- Pas d'animation pendant scroll.
