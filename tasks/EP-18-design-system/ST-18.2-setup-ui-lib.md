## Contexte business

Le choix UI lib impacte la vélocité et la cohérence. **Décision actée (ADR-001)** : `@nuxt/ui` 4.8 (Free + Pro unifiés en open source depuis avril 2026) + Tailwind CSS 4.3 (config CSS-first via `@theme`). Cette story effectue le setup et vérifie que les composants consomment bien les tokens du DS (ST-18.1).

## Description fonctionnelle

ADR `docs/adr/008-ui-lib-choice.md` documentant pourquoi `@nuxt/ui` v4 a été retenu vs alternatives (shadcn-vue, Reka UI / Radix Vue port, PrimeVue, Vuetify). Setup : install `@nuxt/ui` ^4.8, configuration `app.config.ts` (`ui.primary`, `ui.gray`, `ui.notifications`) qui consomme les tokens définis en ST-18.1. Vérification : `<UButton>`, `<UInput>`, `<UModal>`, `<UCommandPalette>`, `<USkeleton>` consomment bien le theme custom et basculent en dark mode automatiquement.

## Cas limites à traiter

- **Override insuffisant @nuxt/ui** : fallback en wrappers custom (`AppButton.vue` enveloppe `UButton` pour cas extrêmes).
- **Conflit Tailwind classes** : @nuxt/ui v4 gère proprement la priorité via `tw-merge`.
- **Composants manquants** (DataTable advanced) : `UTable` couvre 90 %, fallback TanStack Table pour cas complexes.
- **Migration ultérieure** : isolation via wrappers (AppButton, AppInput) pour les cas non-trivials.
- **Bundle size impact** : tree-shaking actif, mesurer après setup.

## Stack UI utilisée

- **@nuxt/ui ^4.8** — Free + Pro unifiés en open source. >70 composants accessibles : `UButton`, `UInput`, `UModal`, `UForm`, `UFormField`, `UCommandPalette`, `UToast`, `UTooltip`, `USkeleton`, `UTable`, `UDropdownMenu`, etc.
- **Tailwind CSS ^4.3** — utility-first, config CSS-first via `@theme` dans `assets/css/main.css`.
- **Reka UI** — sous le capôt de @nuxt/ui pour les primitives a11y (port Vue de Radix UI).
- **Tabler Icons** via `@iconify-json/tabler` — outline only.
- **Vue 3.5** + Nuxt 4.4 + TypeScript 5.6 strict.
