## Contexte business

Les atomes sont les briques fondamentales du design system (Button, Input, Badge, Avatar, Icon, Spinner, Skeleton, Tooltip, Switch, Checkbox, Radio, Tag). Sans cette base, chaque feature réinvente une variante — erreur de cohérence garantie. Cette story livre tous les atomes en étendant les composants `@nuxt/ui` 4.8, avec Storybook 10 + Chromatic pour les variants, states et sizes.

## Description fonctionnelle

Dossier `components/atoms/` avec des wrappers fins autour des composants `@nuxt/ui` v4 (qu'on n'étend que si la story Cursus a un besoin spécifique non couvert par le composant natif) :

- `AppButton.vue` enveloppe `UButton` (variants primary, secondary, ghost, danger ; sizes sm/md/lg ; states loading, disabled)
- `AppInput.vue`, `AppTextarea.vue`, `AppSelect.vue`, `AppCheckbox.vue`, `AppRadio.vue`, `AppSwitch.vue` enveloppent les composants form @nuxt/ui (avec labels et states a11y)
- `AppBadge.vue` enveloppe `UBadge` (variants : default, success, warning, danger, info)
- `AppAvatar.vue` enveloppe `UAvatar` (sizes xs/sm/md/lg/xl, fallback initiales)
- `AppIcon.vue` enveloppe `UIcon` (Tabler Icons via `@iconify-json/tabler`, sizes 16/20/24)
- `AppSpinner.vue`, `AppSkeleton.vue` (autour de `USkeleton`), `AppTooltip.vue`, `AppTag.vue`

Chaque composant : stories Storybook 10 (variants, dark/light), tests unit Vitest 4.1, audit a11y axe-core, snapshot Chromatic.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Button avec ses variants**

```gherkin
Given AppButton.vue (wrapper UButton) existe
When Storybook montre la story
Then 4 variants × 3 sizes × dark+light affichables
And a11y check axe-core passe (focus, contrast, role)
```

**Scénario 2 — Input avec validation**

```gherkin
Given AppInput avec prop :error="true" (utilise UFormField sous le capôt)
When le composant rend
Then la bordure passe danger color (token --color-danger-500)
And aria-invalid="true"
And un slot error/helper s'affiche
```

**Scénario 3 — Avatar fallback**

```gherkin
Given AppAvatar sans src
When le composant rend
Then on voit les initiales (MS pour Mohamed S.)
And background aléatoire basé sur le hash du nom
```

## Cas limites à traiter

- **Button icon-only** : aria-label requis (TypeScript prop required).
- **Input long** : truncate text-overflow ellipsis.
- **Avatar URL 404** : fallback initiales.
- **Tooltip sur mobile** : long-press support (géré par UTooltip).
- **Checkbox indeterminate** : prop supporté.

## Stack UI utilisée

- **@nuxt/ui ^4.8** — composants de base enveloppés : `UButton`, `UInput`, `UTextarea`, `USelect`, `UCheckbox`, `URadioGroup`, `USwitch`, `UBadge`, `UAvatar`, `UIcon`, `USkeleton`, `UTooltip`, `UFormField`.
- **Tailwind CSS ^4.3** — utility-first, tokens via `@theme`.
- **Tabler Icons** via `@iconify-json/tabler` — outline only, sizes 16/20/24px.
- **Storybook 10.4** + `@storybook/vue3-vite` + **Chromatic** pour tests visuels en CI.
- **Vitest 4.1** pour les tests unitaires des wrappers.
- **axe-core** via `@storybook/addon-a11y` pour l'audit a11y des stories.

## Considérations sécurité

- Input : XSS protection v-html proscrit, slot avec sanitize si rich content.
- Sanitize alt text si dynamic.

## Considérations accessibilité

- Tous atomes AA conformes natifs (hérité de @nuxt/ui + Reka UI).
- AAA sur Button + Input (écrans critiques auth).
- Focus visible always (no outline:none).
- ARIA roles corrects (button, textbox, checkbox).
- Touch targets ≥ 44×44 px.

## Considérations performance

- Composants légers (< 5 KB chacun gzip pour le wrapper).
- Tree-shakeable.
- Lazy load Icon component (Tabler Icons : icon par icon via `@iconify-json/tabler`, pas tout le pack).
