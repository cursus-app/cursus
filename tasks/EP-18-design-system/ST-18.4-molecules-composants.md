## Contexte business

Après les atomes (ST-18.3), les molécules orchestrent ces briques dans des unités réutilisables : Card, FormField (label + input + erreur), Alert (notif inline), Banner, Toast, EmptyState, ProgressBar, Stepper, DataTable, SearchInput, DateRangePicker. Sans ce niveau, les pages réimplémentent ces patterns chacune différemment. Cette story livre l'étage « molecules » du design system en s'appuyant sur les composants @nuxt/ui v4 (`UCard`, `UFormField`, `UAlert`, `UToast`, `UTable`, `UProgress`, `USteps`).

## Description fonctionnelle

Dossier `components/molecules/` :

- `AppFormField.vue` (wrapper `UFormField` : label, input slot, error, helper, validation vee-validate)
- `AppCard.vue` (wrapper `UCard` : slots header, body, footer)
- `AppAlert.vue` (wrapper `UAlert` : inline, dismissible, variants success/warning/danger/info)
- `AppBanner.vue` (top-of-page, dismissible, persistant via localStorage)
- `AppToast.vue` (utilise `useToast()` de @nuxt/ui, position bottom-right, queue 3 max)
- `AppEmptyState.vue` (illustration + titre + description + CTA)
- `AppProgressBar.vue` (wrapper `UProgress` linear + circular)
- `AppStepper.vue` (wrapper `USteps` : multi-step forms)
- `AppDataTable.vue` (wrapper `UTable` : tri, filtre, pagination ; TanStack Table en fallback complexe)
- `AppSearchInput.vue` (wrapper `UInput` + Cmd+K hint via `UKbd`)
- `AppDateRangePicker.vue` (wrapper `UInputMenu` + calendrier custom)

Chaque molécule : story Storybook 10, tests unit Vitest 4.1, audit a11y.

## Critères d'acceptation (Gherkin)

**Scénario 1 — FormField avec erreur**

```gherkin
Given AppFormField label "Email", error "Invalide" (utilise UFormField + vee-validate)
When rendu
Then label lié à input via for=id
And aria-describedby pointe vers le message d'erreur
And la border devient danger (token --color-danger-500)
```

**Scénario 2 — EmptyState**

```gherkin
Given AppEmptyState avec illustration, title, description, action
When affiché sur une page vide
Then utilisateur voit message empathique + CTA clair (UButton)
And l'illustration a alt textuel équivalent
```

**Scénario 3 — DataTable tri**

```gherkin
Given AppDataTable (UTable) avec colonnes sortable
When je clique sur une colonne
Then le tri se fait
And aria-sort="ascending" est annoncé
And l'URL contient le param sort
```

## Cas limites à traiter

- **Toast emergency** : queue de toasts (max 3 simultanés).
- **DataTable vide** : EmptyState intégré.
- **Card sans slot** : pas de render (évite empty divs).
- **DateRangePicker timezone** : tout en local user, UTC en API.
- **Banner persistante** : stockage localStorage du dismiss.

## Stack UI utilisée

- **@nuxt/ui ^4.8** — composants utilisés : `UCard`, `UFormField`, `UAlert`, `UToast` (via `useToast()`), `UTable`, `UProgress`, `USteps`, `UInput`, `UInputMenu`, `UKbd`.
- **Tailwind CSS ^4.3** — utility-first, tokens via `@theme`.
- **Formulaires** : vee-validate ^4 + `@vee-validate/zod` + Zod ^3.23 (schéma symétrique client/serveur).
- **TanStack Table** ^9 — fallback pour `AppDataTable` quand les besoins dépassent `UTable` (virtualisation, group by, pivot).
- **Tabler Icons** via `@iconify-json/tabler` — outline only.
- **Storybook 10.4** + **Chromatic** pour tests visuels.
- **Vitest 4.1** pour les tests unitaires.

## Considérations sécurité

- DataTable : sanitization des données de tri (pas d'injection).
- DateRangePicker : validation bornes.
- Toast : pas de HTML injecté sans sanitize.

## Considérations accessibilité

- FormField : association label-input correcte.
- DataTable : `<table>` sémantique, `<th scope>`, `aria-sort`.
- Toast : `aria-live="polite"`.
- EmptyState : description textuelle alternative.
- Stepper : navigation clavier.

## Considérations performance

- DataTable : virtualisation > 100 rows (TanStack Table).
- Toast : limit 3 simultanés pour pas surcharger.
- ProgressBar : SVG inline (pas de blocage).
