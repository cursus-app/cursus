## Contexte business

Les **design tokens** sont la fondation du design system Premium (axe Premium 1). Sans tokens centralisés (couleurs, typo, spacings, radius, motion), chaque composant réinvente sa propre cohérence et le dark mode devient impossible. Cette story implemente la couche de tokens citée en design-system §1 sur **Tailwind CSS 4.3** (config CSS-first via `@theme`).

## Description fonctionnelle

Fichier `assets/css/main.css` exposant les tokens via le bloc Tailwind 4.3 `@theme` : couleurs sémantiques (`--color-bg-base`, `--color-fg-default`, `--color-accent-base`, etc.), typo (`--font-sans`, `--font-mono`), radius (`--radius-sm`, `--radius-md`), shadows, durations motion, z-index, spacings (4-pt scale). Mode dark via `@media (prefers-color-scheme: dark)` + classe `.dark` override (priorité manuelle).

Plus de `tailwind.config.js` obligatoire : la config est 100 % CSS, lue par Tailwind 4.3 + `@nuxt/ui` 4.8. Documentation `docs/design/tokens.md`.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Tokens light mode**

```gherkin
Given assets/css/main.css avec bloc @theme est chargé
When je consulte le DOM en light mode
Then les CSS vars existent : --color-bg-base = oklch(...), etc.
And un composant qui utilise text-fg-default résoud bien vers la var
```

**Scénario 2 — Tokens dark mode**

```gherkin
Given la classe .dark est sur <html>
When je consulte
Then les CSS vars dark override : --color-bg-base = oklch(0.15 ...)
And les composants se basculent automatiquement
```

**Scénario 3 — Tokens consommés par @nuxt/ui v4**

```gherkin
Given @nuxt/ui 4.8 est installé
When j'utilise <UButton color="primary">
Then la couleur primary mappe vers --color-primary-500 défini dans @theme
And le dark mode fonctionne automatiquement (mêmes classes)
```

## Cas limites à traiter

- **Couleur contraste insuffisant** : test automatique WCAG sur paires de tokens.
- **Nouveau token à ajouter** : process documente dans `docs/design/tokens.md`.
- **Token légacy en code** : codemod fournis pour migrer (ex : `bg-white` → `bg-bg-base`).
- **prefers-color-scheme** : fallback automatique si pas de classe `.dark`.
- **Performance** : pas d'overhead CSS var (1 indirection native).

## Stack UI utilisée

- **Tailwind CSS ^4.3** — config CSS-first via `@theme` dans `assets/css/main.css`, plus de `tailwind.config.js` obligatoire. Tokens en CSS vars sur l'espace colorimétrique OKLCH.
- **@nuxt/ui ^4.8** — consomme les tokens via son système de theming (`app.config.ts` mappe `primary`, `gray`, `success`, etc. vers les CSS vars).
- **Tabler Icons** via `@iconify-json/tabler` — outline only, sizes 16/20/24px.
