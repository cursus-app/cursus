## Contexte business

Un écran vide ("aucun cursus créé", "aucune cohorte démarrée", "aucun stagiaire à risque") est un moment décisif de l'UX : soit l'utilisateur est perdu et abandonne, soit on en profite pour lui montrer la valeur du produit et l'orienter vers la prochaine action. Les empty states illustrés livrent ce moment proprement sur 10 écrans clés du produit. C'est un marqueur direct du pilier Premium « finition design » (voir 10-design-system §3.4 et 01-vision axe Premium-1).

Sans cette story, chaque vide ressemble à un bug ; avec, chaque vide devient une opportunité pédagogique (« crée ton premier cursus » avec illustration sobre + CTA).

## Description fonctionnelle

Livrer **10 empty states cohérents** réutilisant la molécule `AppEmptyState.vue` (ST-18.4), chacun avec : illustration SVG sobre (style line-art monochrome adapté au thème via tokens), titre empathique, description courte (1-2 phrases), CTA primaire (`UButton`), lien d'aide secondaire. Les illustrations sont produites in-house en SVG (palette tokens du DS via `@theme`) ou tirées d'un set libre re-stylisé (unDraw monochrome).

Les 10 écrans cibles : (1) Liste cursus vide, (2) Liste cohortes vide, (3) Cohorte sans stagiaire, (4) Stagiaire sans soumission, (5) Portfolio vide, (6) Alertes — aucune alerte active, (7) Notifications — boîte vide, (8) Quiz — aucun quiz disponible, (9) Recherche sans résultat, (10) Audit log filtré à zéro.

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Premier cursus à créer
Given un formateur vient de créer son compte
When il arrive sur /cursus
Then l'écran affiche une illustration sobre, le titre "Aucun cursus pour le moment"
And une description "Crée ton premier cursus pour commencer à former."
And un CTA primaire UButton "Créer un cursus" qui ouvre le builder
And un lien secondaire "Comment fonctionne un cursus ?"
```

```gherkin
Scénario 2 — Recherche sans résultat
Given un utilisateur tape "toto42" dans la UCommandPalette
When aucun résultat ne remonte
Then l'écran affiche une illustration "loupe", le titre "Aucun résultat pour \"toto42\""
And une description "Essaie un autre mot-clé ou parcours la liste complète."
And un CTA "Effacer la recherche"
```

```gherkin
Scénario 3 — Dark/light cohérent
Given le thème dark est actif
When un empty state est affiché
Then les couleurs de l'illustration utilisent les tokens sémantiques (--color-surface-muted, --color-fg-subtle)
And le contraste reste ≥ 4.5:1 sur le texte
And aucune couleur n'est codée en dur
```

## Cas limites à traiter

- **Vide temporaire (chargement)** : ne pas afficher l'empty state pendant que la requête est `pending` — afficher le skeleton (ST-18.7) puis basculer en empty state si la réponse est `[]`.
- **Vide après filtre** : message différent (« Aucun résultat pour ces filtres ») avec CTA « Réinitialiser les filtres ».
- **Vide après erreur** : ne pas confondre avec un error state — afficher un message d'erreur dédié (UAlert variant danger) si la requête a échoué.
- **Droits insuffisants** : empty state spécifique « Tu n'as pas accès à cette section » avec lien vers l'aide rôle.
- **Empty + onboarding non terminé** : prioriser un onboarding teaser plutôt que l'empty state standard.

## Non-goals

- Pas d'illustrations animées (statique SVG).
- Pas d'A/B test sur le copywriting au MVP (peut venir en v1.x).
- Pas d'empty states sur les composants atomiques internes (input vide, etc.).
- Pas d'illustration custom par cohorte ou par cursus.

## Tests à écrire

- **Unit Vitest 4.1** : `AppEmptyState` reçoit le bon variant, rend bonnes clés i18n, CTA déclenche l'event attendu.
- **A11y axe-core** : 0 violation critique, illustration a un `role="img"` + `aria-label` ou est marquée `aria-hidden` si décorative.
- **Visual Chromatic** : snapshots des 10 variants en dark + light.
- **E2E Playwright** : créer un compte neuf → vérifier que `/cursus` affiche bien l'empty state "Aucun cursus".
- **i18n** : tests garantissant qu'aucune string n'est codée en dur (cf ST-19.6).

## Observabilité

- **Logs** : event `empty_state.viewed` (info) avec payload `{ variant, route, user_role }` — sert à mesurer quels empty states sont vus en pratique.
- **Métriques** : compteur `cursus.empty_state.views.total` labellisé par `variant`. Taux de clic sur CTA : `cursus.empty_state.cta_clicked.total`.
- **Insight** : si le taux de clic CTA d'un empty state est < 5 % sur 100 vues → revoir copy/illustration.

## Considérations sécurité

- Pas de données PII dans le DOM de l'empty state (purement statique).
- SVGs servis depuis le bundle (pas d'XHR externe), donc pas de surface d'attaque SSRF.

## Considérations accessibilité

- Illustration décorative → `aria-hidden="true"`. Illustration porteuse de sens → `role="img"` + `aria-label` traduit.
- Titre `<h2>` correctement hiérarchisé dans la page.
- CTA `UButton` navigable au clavier, focus visible (token `--ring-focus`).
- Texte ≥ 16 px, contraste ≥ 4.5:1, respect `prefers-reduced-motion` (pas d'animation shimmer sur l'illu).

## Considérations performance

- Chaque SVG < 5 KB inline, total payload illustrations < 50 KB.
- Pas de polices custom pour l'empty state (réutilise la police globale).
- Lazy-load impossible (l'empty state est dans le viewport), donc inline SVG préféré à `<img>` pour éviter une requête HTTP.
