## Contexte business

Le dark mode est un standard moderne et un signal premium. Avec les tokens (ST-18.1) en place via Tailwind 4.3 `@theme` et @nuxt/ui 4.8 qui supporte nativement `useColorMode` (via `@vueuse/nuxt`), l'implémentation est triviale ; ce qui ne l'est pas, c'est l'UX (transition fluide, persistance préférence, respect prefers-color-scheme). Cette story finalise l'expérience.

## Description fonctionnelle

Toggle UI (icon sun/moon dans le header) avec 3 modes : `light`, `dark`, `system` (suit `prefers-color-scheme`). Utilise `useColorMode()` de `@vueuse/nuxt` (couplage natif avec @nuxt/ui v4). Préférence stockée en localStorage (utilisateur anonyme) + colonne `user_preferences.theme` Prisma (authentifié). Au load, le thème est appliqué avant 1ᵉ paint via le module Nuxt color-mode (évite FOUC). Transition crossfade 250 ms (cf. design-system §5).

## Critères d'acceptation (Gherkin)

**Scénario 1 — Toggle**

```gherkin
Given un utilisateur sur le site
When il clique sur le toggle (UButton avec UIcon sun/moon)
Then le thème bascule de light → dark via useColorMode()
And la transition dure 250 ms (crossfade CSS)
And la préférence persiste localStorage + DB si authentifié
```

**Scénario 2 — Suivre système**

```gherkin
Given un user choisit "system"
When son OS bascule dark
Then l'app bascule automatiquement
And un MediaQuery listener est actif (géré par useColorMode)
```

**Scénario 3 — Multi-device sync**

```gherkin
Given un user authentifié change la préf sur device A
When il se connecte sur device B
Then la même préf s'applique (lue depuis DB via Prisma)
```

## Cas limites à traiter

- **FOUC au 1ᵉ load** : module Nuxt color-mode pose la classe avant <body>.
- **prefers-reduced-motion** : pas de crossfade animation.
- **localStorage indisponible (Safari incognito)** : fallback session-only.
- **Conflict localStorage vs DB** : DB wins quand auth.
- **Image-only contenu** : adapter selon thème (different src).

## Sous-tâches techniques

- **TT-18.5.1** — Composable `useTheme()` (wrap `useColorMode()` + sync DB pour user auth)
- **TT-18.5.2** — Composant `ThemeToggle.vue` (UButton + UIcon sun/moon)
- **TT-18.5.3** — Module Nuxt color-mode actif (set class avant 1ᵉ paint via plugin Nitro)
- **TT-18.5.4** — MediaQuery listener pour mode "system" (géré par useColorMode)
- **TT-18.5.5** — Migration Prisma : ajouter `theme` à `UserPreference` (`prisma/schema.prisma`)
- **TT-18.5.6** — Endpoint `PUT /api/me/preferences/theme` (Nitro + Prisma)
- **TT-18.5.7** — Tests E2E Playwright : toggle persistant
- **TT-18.5.8** — Adaptation des images si besoin (logos darken)

## Stack UI utilisée

- **@nuxt/ui ^4.8** — composants `UButton`, `UIcon` (Tabler `tabler:sun`, `tabler:moon`, `tabler:device-laptop`).
- `@vueuse/nuxt` ^12 — `useColorMode()` (couplage natif avec @nuxt/ui v4).
- **Tailwind CSS ^4.3** — dark mode via classe `.dark` sur `<html>` + `@theme` tokens.
- **Prisma 7.8** — modèle `UserPreference` (colonne `theme`).
- **Module Nuxt color-mode** — actif par défaut via `@nuxt/ui` v4, pas de FOUC.

## Dépendances

- **Bloqué par** : ST-18.1 (tokens)

## Non-goals

- Pas de thèmes custom (high contrast, sepia) au MVP.
- Pas de scheduling auto ("dark après 18h").
- Pas de personnalisation per-cohorte.

## Tests à écrire

- **Unit Vitest** : `useTheme()` résolution priority (DB > localStorage > system).
- **E2E Playwright** : toggle persistant cross-reload.
- **A11y** : toggle accessible clavier, aria-label clair, role="switch" ou button.
- **Visual Chromatic** : dark + light de tous écrans clés.

## Observabilité

- **Logs** : `user.preference.theme_changed` (info, payload : `from`, `to`)
- **Métriques** : Compteur `cursus.theme.changes.total` labellisé par `to`
- **Plausible** : event "theme_toggled"

## Considérations sécurité

- Pas de PII dans la préf.
- Stockage DB scopé à l'utilisateur (RLS Supabase + rôle Prisma `cursus_app`).

## Considérations accessibilité

- Toggle : aria-label "Basculer en mode sombre", role="switch" ou button.
- prefers-reduced-motion respecté.
- Contraste AA testé both modes.
- Annonce du nouveau thème discrètement.

## Considérations performance

- Module color-mode gère le no-FOUC (< 200 bytes inline).
- Crossfade GPU-accelerated.
- Pas de re-render JS (CSS vars seules).
