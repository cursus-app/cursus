## Contexte business

Microsoft Teams est la plateforme dominante en entreprise traditionnelle (banques, grandes ESN, secteur public). Les cohortes de reconversion en entreprise sont souvent sous Teams. Sans ce connecteur, Cursus se ferme à ce segment. Symétrique à Slack et Discord.

Business value : ouverture au segment B2B entreprise. Couverture quasi-totale du marché français en collaboration (Slack + Teams \~ 80 %).

## Description fonctionnelle

- Connecteur Teams via Incoming Webhook (Office 365 Connector).
- Mêmes events que Slack/Discord.
- Format : Adaptive Card (JSON v1.5) avec titre, sections, actions (link).
- Throttling : Teams supporte \~4 messages / sec ; cap 2/s pour sécurité.
- Retry + audit + réutilise infrastructure ST-22.1.

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Adaptive Card sur soumission
Given une cohorte avec webhook Teams configuré
When un event submission.failed arrive
Then un Adaptive Card arrive dans le canal Teams
And il a header rouge, titre "Soumission échouée", contexte (nom stagiaire, module)
And action "Voir le rapport" lien externe
```

```gherkin
Scénario 2 — Configuration
Given l'admin colle un webhook Teams
When il teste
Then un POST de vérification est fait
And succès = badge vert, échec = message clair
```

```gherkin
Scénario 3 — Teams en panne
Given Teams renvoie 503
When retry exponentiel
Then 3 tentatives, puis marquage failed
And log + alerte admin si > 10 échecs en 1 h
```

## Cas limites à traiter

- **Card schema version trop récent** : tester sur instance Teams réelle, fallback v1.0 si besoin.
- **Card too long** (Teams limit \~ 28 KB) : tronquer + lien Cursus.
- **Actions limitées** : Teams accepte `Action.OpenUrl` mais pas tout. Stay simple.
- **Webhook expiré** : Teams les invalide parfois → détecter 410 et alerter admin pour re-configurer.
- **Tenant suspendu** : 403 → disable intégration.

## Non-goals

- Pas de bot Teams.
- Pas de tabs Teams app.
- Pas de meeting integration.
- Pas de adaptive card avec inputs (juste affichage + lien).

## Tests à écrire

- **Unit Vitest** : `formatAdaptiveCard` valide selon schema Adaptive Card.
- **Integration** : event → Teams mocké → payload validé.
- **Security** : hostname strict.
- **A11y** : Teams gère le rendu.

## Observabilité

- **Logs** : `integrations.teams.sent` (info).
- **Métriques** : compteur, taux d'échec, latence.
- **Alerte** : taux d'échec > 10 % sur 1 h.

## Considérations sécurité

- URL chiffrée.
- Hostname strict (`*.webhook.office.com`).
- RLS.

## Considérations accessibilité

- Adaptive Card respecte l'a11y native Teams.
- Color + symbol pour status (pas couleur seule).

## Considérations performance

- Job async, cap 2/s.
- Cible message < 5 s p95.
