## Contexte business

Le RGPD impose à tout responsable de traitement de fournir au sujet (stagiaire, formateur) un export complet de ses données personnelles sur demande, sous 30 jours, dans un format réutilisable. Sans cette story, Cursus est en violation du RGPD (article 20 — portabilité). Cela touche aussi la conformité à venir des organisations clientes (ISO 27001, SOC 2).

La story livre un parcours self-service : l'utilisateur déclenche son propre export depuis ses paramètres, reçoit un email avec un ZIP signé, sans dépendre de l'admin.

## Description fonctionnelle

Dans "Mon profil > Confidentialité", un bouton "Demander un export de mes données". Au clic, un job Inngest est enqueué. Le job collecte toutes les données personnelles (profil, soumissions, livrables, quiz, badges, certificats, alertes, notifications, audit log filtré sur l'acteur=user) et produit un ZIP avec :

- `data.json` : structure JSON normalisée (toutes les tables)
- `submissions/<id>/` : copie des fichiers livrables (s'ils sont stockés en Storage)
- `certificates/<id>.pdf` : copie des certificats émis
- `README.txt` : schéma + index

Le ZIP est uploadé dans Supabase Storage avec une URL signée 7 jours envoyée par email.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Demande d'export par un stagiaire**

```gherkin
Given un stagiaire connecté
When il navigue dans "Mon profil > Confidentialité"
And il clique sur "Demander un export de mes données"
Then un job Inngest "gdpr.export" est enqueué
And l'UI affiche "Ta demande est en cours. Tu recevras un email sous 24h"
And un événement "gdpr.export.requested" est tracé
And l'utilisateur ne peut pas relancer une demande pendant 24h (anti-spam)
```

**Scénario 2 — Réception de l'export**

```gherkin
Given un job d'export termine
When le ZIP est uploadé dans Storage
Then un email Resend est envoyé avec un lien signé valide 7 jours
And l'événement "gdpr.export.delivered" est tracé
And le ZIP contient data.json + submissions/ + certificates/ + README.txt
```

**Scénario 3 — Contenu de l'export d'un formateur**

```gherkin
Given un formateur demande son export
When le job s'exécute
Then le ZIP contient son profil et ses contributions (cursus créés, commentaires, override)
And ne contient PAS les données personnelles des stagiaires (séparation des sujets)
```

## Cas limites à traiter

- **Demande répétée < 24h** : refus avec message "Tu as déjà demandé un export il y a Xh. Réessaie demain."
- **Job échoue (DB indisponible)** : retry 3× par Inngest, puis email d'excuse + alerte ops.
- **Volume de fichiers énorme (> 500 Mo)** : split en multi-archive `.zip.001`, `.zip.002`, etc.
- **Utilisateur supprimé pendant la génération** : job annulé, pas d'email envoyé.
- **Données croisées (commentaire sur un autre user)** : pseudonymisation des autres acteurs (UUID + initial).
- **Conformité format** : JSON-LD respecte un schéma documenté dans `README.txt`.

## Non-goals

- Pas d'export d'analytics agrégés sur la cohorte du stagiaire (autres utilisateurs).

- Pas de personnalisation du contenu de l'export (tout ou rien).

## Tests à écrire

- **Unit** :

  - `pseudonymizeForeignActors()` : initiales correctement extraites, UUID préservé
  - `streamZipBuilder()` : ZIP valide vérifié avec `unzip -l`
  - Rate limit : 2 demandes < 24h → 2e refusée

- **Integration** :

  - Job complet sur user seed → ZIP contient \~12 tables + fichiers
  - Vérifier qu'un export de stagiaire ne contient PAS le profil d'un autre stagiaire de la cohorte
  - Email Resend mocké reçoit le lien signé

- **E2E (Playwright)** :

  - Stagiaire clique → email reçu (mock) → télécharge ZIP → ouvre data.json

- **A11y (axe-core)** :

  - Page privacy accessible clavier, bouton avec label clair
  - Confirmation de demande annoncée via `aria-live`

- **Security** :

  - URL signée expire bien à 7 jours
  - Tentative d'accès à l'export d'un autre user → 403
  - Pas de fuite de données croisées (test sur seed avec 2 stagiaires)

- **Performance** :

  - Export d'un user avec 50 submissions en < 60 s

## Observabilité

- **Logs** :

  - `gdpr.export.requested` (info, payload : `user_id`)
  - `gdpr.export.completed` (info, payload : `job_id`, `bytes`, `duration_ms`)
  - `gdpr.export.failed` (error)

- **Métriques** :

  - Compteur `cursus.gdpr.exports.requested.total`
  - Histogram `cursus.gdpr.export.duration.seconds`

- **Alertes** : taux d'échec > 10 % sur 7 jours (problème de pipeline).

## Considérations sécurité

- Rate limit strict (1 demande/24h/user) via Upstash Redis.
- URL signée HMAC + expiration 7 jours.
- ZIP chiffré au repos par Supabase Storage.
- Pseudonymisation des autres acteurs pour ne pas créer une fuite croisée.
- Audit log de toute demande et livraison.
- Pas de logs des contenus exportés (PII).

## Considérations accessibilité

- Bouton avec label clair, focus visible.
- État de la demande annoncé via `aria-live="polite"`.
- Historique des demandes : tableau accessible.
- Email RGPD rédigé dans la langue de l'utilisateur (cohérent avec ST-19).

## Considérations performance

- Streaming ZIP (pas de buffer mémoire global).
- Index Prisma `@@index([userId])` sur tous les models consultés.
- Cible : export user avec 50 submissions < 60 s.
- Job plafonné à 3 concurrents par instance pour limiter charge DB.
