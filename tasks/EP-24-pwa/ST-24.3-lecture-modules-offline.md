## Contexte business

Un stagiaire dans le métro veut relire son module en cours — sans connexion. "Offline lite" = lecture des modules déjà visités reste possible quand la connexion tombe. Ce n'est pas un mode offline complet (pas de soumission, pas d'édition), juste la lecture du contenu déjà vu.

Business value : améliore drastiquement l'usage mobile en déplacement. Marqueur premium d'app pro.

## Description fonctionnelle

- Quand un user consulte un module en ligne, son contenu (markdown + métadonnées) est caché en IndexedDB pour 7 jours.
- En offline, la liste "Mes modules récents" est accessible.
- La page module se charge depuis IndexedDB si offline.
- Banner persistant "Mode hors-ligne — lecture seule".
- Soumission / quiz désactivés avec message clair.
- Sync au retour en ligne : flush cache, fetch les versions récentes.

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Lecture offline
Given l'user a visité le module M en ligne hier
When il revient hors-ligne aujourd'hui
Then le module M est lisible entièrement
And un banner discret "Hors-ligne — lecture seule" est affiché
And les actions (soumission, quiz) sont désactivées avec tooltip
```

```gherkin
Scénario 2 — Liste modules récents
Given l'user a visité 5 modules ces 7 jours
When il ouvre /modules hors-ligne
Then les 5 modules apparaissent (cachés)
And les autres sont grisés avec mention "Indisponible hors-ligne"
```

```gherkin
Scénario 3 — Retour en ligne
Given l'user revient en ligne
When la page est réaffichée
Then un fetch réel remplace le cache si version > stale
And le banner offline disparaît
And les actions se réactivent
```

## Cas limites à traiter

- **IndexedDB plein** : LRU eviction, dernier accès prioritaire.
- **Cache stale > 7 jours** : afficher avertissement "Contenu peut être obsolète".
- **Module modifié par formateur entre temps** : sync au retour en ligne, notif si majeur.
- **Quiz / soumission tentés offline** : modal claire "Disponible uniquement en ligne. Tu peux préparer dans un brouillon."
- **iframe / vidéo externes** : ne sont pas cachées (limit déclarée), message gracieux.
- **App fraîchement installée (cache vide)** : page offline avec liste vide.

## Sous-tâches techniques

- **TT-24.3.1** — Helper `cacheModule(moduleId)` qui stocke `{ id, title, content, lastSync }` dans IndexedDB.
- **TT-24.3.2** — Lecture priorité : `network-first` si online, `cache-only` si offline.
- **TT-24.3.3** — Composable `useOfflineStatus()` (Online/Offline event).
- **TT-24.3.4** — Composant `<OfflineBanner />` persistant tant que offline.
- **TT-24.3.5** — Désactivation conditionnelle des actions (boutons disabled + tooltip).
- **TT-24.3.6** — Indicateur stale ("Mis à jour il y a 3 jours") sur les modules cachés.
- **TT-24.3.7** — Politique de purge auto (LRU + 7 jours TTL).
- **TT-24.3.8** — Tests offline mode Playwright.

## Dépendances

- **Bloqué par** : ST-24.2 (service worker), ST-05.\* (page module existe).

## Non-goals

- Pas de cache des PDF / vidéos volumineuses.
- Pas d'édition offline (lecture seule).
- Pas de cache du dashboard / reporting.
- Pas de search offline.

## Tests à écrire

- **Unit Vitest** : `cacheModule` stocke correctement, LRU eviction.
- **Integration** : DevTools offline → module lit depuis cache.
- **E2E Playwright** : online → visiter module → offline → relire → banner.
- **A11y axe-core** : banner et indicateurs accessibles.
- **i18n** : banner traduit.
- **Performance** : lecture offline < 200 ms.

## Observabilité

- **Logs** : `offline.read` (info) avec `{ module_id, source: 'cache'|'network', stale_days }`.
- **Métriques** : compteur reads offline, taille IndexedDB moyenne.
- **Insight** : modules les plus consultés offline.

## Considérations sécurité

- Cache isolé par origine (même domaine seulement).
- Pas de PII sensible dans IndexedDB (juste contenu pédagogique).
- Purge à la déconnexion (logout efface cache).
- RLS respecté à la lecture initiale.

## Considérations accessibilité

- Banner `role="status"` (info, pas urgent).
- Boutons désactivés avec `aria-disabled="true"` + tooltip.
- Message clair quand contenu absent du cache.

## Considérations performance

- IndexedDB cap (cible 20 MB par user).
- Lecture cache < 100 ms.
- Pas d'impact sur le boot online.
- Cible : transition online → offline transparente.
