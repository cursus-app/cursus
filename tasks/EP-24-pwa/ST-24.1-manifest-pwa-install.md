## Contexte business

Les stagiaires consultent souvent leur module dans le métro ou pendant une pause sans wifi fiable. Une PWA installée sur l'écran d'accueil mobile transforme Cursus en "app" : icône dédiée, fullscreen, splash. C'est le ticket d'entrée de l'épic Offline (EP-24).

Business value : marqueur premium d'app moderne, base pour l'offline (ST-24.2+), rétention mobile +20 %.

## Description fonctionnelle

- Fichier `public/manifest.webmanifest` complet : `name`, `short_name`, `start_url`, `display: 'standalone'`, `theme_color`, `background_color`, `icons` (192, 512, maskable).
- `<link rel="manifest" href="/manifest.webmanifest">` dans le `<head>`.
- Meta `<meta name="theme-color" content="...">` adapté au mode (dark/light).
- Splash screen iOS (spécifique apple-touch).
- Install prompt UI : banner discret "Installer Cursus sur ton téléphone" affiché après 3 visites, dismissible 30 j.
- Gestion event `beforeinstallprompt`, exposition via composable.

## Cas limites à traiter

- **iOS Safari** : pas d'event `beforeinstallprompt` — fournir un guide d'install manuel ("Partager > Sur l'écran d'accueil").
- **App déjà installée** : event `appinstalled` cache le banner.
- **Brave / DDG** : peuvent bloquer install prompt — gracieux fallback.
- **Dark mode change** : theme_color dynamic n'est pas standard — prendre celle du mode actif au boot.
- **Manifest invalide** : page degrade en mode classique, pas de crash.

## Considérations sécurité

- Manifest serté en HTTPS uniquement.
- Icônes : pas de leak de métadonnées (strip EXIF).
- CSP : ne pas casser le PWA install.

## Considérations accessibilité

- Banner install : navigable au clavier, dismiss focusable.
- `aria-label` explicite sur les boutons.
- Respect prefers-reduced-motion.
- Annonce du switch standalone.

## Considérations performance

- Manifest < 5 KB.
- Icônes optimisées (svgo, sharp).
- Pas d'impact sur LCP / FID.
- Banner mounté lazy (après 3 visites).
