## Contexte business

Feed cohorte = motivation collective sans compétition toxique. **Positif uniquement** (pas d'échecs visibles) — anti-pilier respecté. Feature premium.

Valeur livrée : feed des 10 derniers événements positifs de la cohorte (validation, badge, capstone en avance), opt-out possible.

## Description fonctionnelle

Widget sur le dashboard stagiaire. Vue matérialisée `cohorte_feed`. Feed s'auto-met à jour. Désactivable dans les préférences.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Feed positif**

```gherkin
Given un stagiaire ouvre son dashboard
When le feed cohorte se charge
Then il voit les 10 derniers événements positifs de sa cohorte :
  - X a validé module Y
  - X a débloqué badge Z
  - X a livré son capstone en avance
And aucun événement négatif (échec, retard)
```

**Scénario 2 — Opt-out**

```gherkin
Given un stagiaire désactive le feed dans ses préférences
Then le widget n'apparaît plus
```

**Scénario 3 — Empty state**

```gherkin
Given un stagiaire seul dans la cohorte
Then le feed vide affiche un message empathique
```

## Cas limites à traiter

- **Stagiaire seul dans la cohorte** : feed vide avec message empathique.
- **Stagiaire désactivé mid-cohorte** : ses événements passés anonymisés.
- **Feed > 100 événements** : pagination.

## Non-goals (hors scope)

- Pas d'événement négatif (anti-pilier).
- Pas de likes / commentaires.

## Tests à écrire

- **E2E (Playwright)** : feed se met à jour temps réel.
- **Unit** : seulement événements positifs.
- **Security** : feed scopeé à la cohorte du stagiaire.
