## Contexte business

Gamification légère : XP pour motiver le stagiaire à progresser sans tomber dans le piège des leaderboards toxiques. Feature premium.

Valeur livrée : attribution automatique d'XP à chaque validation, toast "+150 XP" en temps réel, jauge mensuelle pour objectif perso.

## Description fonctionnelle

Event handler `submission.validated` qui ajoute les XP configurés sur le module au compteur stagiaire. Toast temps réel. Jauge mensuelle progresse. Toast spécial si objectif atteint.

## Critères d'acceptation (Gherkin)

**Scénario 1 — XP à la validation**

```gherkin
Given une submission est validée
When l'événement est traité
Then les XP configurés sur le module sont ajoutés au compteur stagiaire
And un toast "+150 XP" s'affiche en temps réel
And la jauge mensuelle progresse
```

**Scénario 2 — Objectif mensuel atteint**

```gherkin
Given le stagiaire atteint son objectif mensuel
Then un toast spécial s'affiche
And l'objectif est marqué atteint en DB
```

**Scénario 3 — Override préserve XP**

```gherkin
Given une submission est validée via override harnais
Then les XP s'appliquent normalement
```

## Cas limites à traiter

- **Override harnais** : attribue les XP normalement.
- **Revalidation après échec** : pas de double XP (idémpotent par submission_id).
- **Révocation de validation** : XP retirés.
- **Nouvelle période mensuelle** : jauge reset.
