## Contexte business

Premier moment de vérité pour le stagiaire. Sans onboarding interactif, il atterrit sur une UI inconnue et l'abandon est garanti dans les 5 min. Différenciateur fort vs un Trello plat. Feature premium qui améliore le taux de rétention.

Valeur livrée : prise en main en 2 min via un tour guidé de 7 étapes, skippable, ré-ouvrable depuis l'aide.

## Description fonctionnelle

Tour guidé en 7 étapes au premier login (lib Driver.js ou Shepherd.js). Skippable. Ré-ouvrable depuis Help → Tour guidé. Perséverance via table `user_tours_completed`. Respect `prefers-reduced-motion`.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Tour au premier login**

```gherkin
Given un stagiaire se connecte pour la 1ʳᵉ fois
When il atterrit sur "Cette semaine"
Then le tour démarre automatiquement avec étape 1 ("Voici ton parcours")
And il peut Skip à tout moment
And la complétion est enregistrée en DB
```

**Scénario 2 — Pas de re-démarrage**

```gherkin
Given il a complété ou skippé le tour
When il revient
Then le tour ne redémarre pas automatiquement
And il peut le relancer depuis Help → Tour guidé
```

**Scénario 3 — Respect reduced motion**

```gherkin
Given prefers-reduced-motion est actif
When le tour démarre
Then les animations sont désactivées (transitions instantanées)
And le tour reste fonctionnel
```

## Cas limites à traiter

- **Élément cible du tour pas encore monté** (race condition) : retry 3× puis skip cette étape.
- **Tour qui pointe vers un élément invisible** (sidebar fermée) : ouvrir l'élément automatiquement.
- **Écran mobile** : adapter tooltip positioning.
- **Resize fenêtre pendant tour** : repositionner.

## Observabilité

- **Logs** : `tour.started`, `tour.completed`, `tour.skipped` (avec step_index).
- **Métriques** : taux de complétion du tour (cible > 60%), étape de skip la plus fréquente.
- **Alertes** : si étape X est skipée par > 50% des users = revoir cette étape.

## Considérations sécurité

- Pas de PII dans la config tour.
- Persistence côté serveur (pas localStorage manipulable).

## Considérations accessibilité

- Tooltips : `role="dialog"`, focus piege, ESC ferme.
- Navigation clavier : Tab + Enter + ESC.
- Annonce lecteur d'écran de l'étape actuelle.
- Contraste 4.5:1 sur l'overlay.
- Respect `prefers-reduced-motion`.

## Considérations performance

- Lazy load la lib tour (uniquement si pas encore complété).
- Pas de blocage du LCP.

## Definition of Done spécifique
