## Contexte business

Pour piloter Cursus comme produit, il faut mesurer son adoption et son usage : combien d'utilisateurs actifs, quels parcours convertissent, où le drop-off se produit. Plausible (cookie-less, RGPD-friendly) est le choix par défaut ; PostHog (self-hosted) en option pour replay et feature flags. Sans cette story, les décisions produit sont aveugles.

## Description fonctionnelle

Intégration Plausible.io via script <script src=plausible> + endpoint custom events `plausible('event_name', { props })`. Pages auto-trackées. Événements custom : `submission_created`, `harness_validated`, `capstone_submitted`, `certificate_issued`, `feature_flag_activated`. Pas de tracking de PII (emails). Dashboard Plausible accessible aux admins via SSO ou compte partagé. Documentation des métriques surveillées.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Pageview tracké**

```gherkin
Given un visiteur charge la home page
When le script Plausible se charge
Then un événement "pageview" est envoyé à Plausible
And aucun cookie n'est posé (cookie-less)
```

**Scénario 2 — Événement custom**

```gherkin
Given un stagiaire soumet un livrable
When la soumission est créée
Then l'événement "submission_created" est envoyé avec props {module_id, attempt_n}
And le dashboard Plausible affiche la métrique en temps quasi-réel
```

**Scénario 3 — Funnel de conversion**

```gherkin
Given les événements signup_started → signup_completed → first_submission existent
When un admin ouvre le dashboard funnels
Then le taux de conversion est visible
And il peut filtrer par cursus et période
```

## Cas limites à traiter

- **AdBlocker bloque Plausible** : OK, c'est mesuré (offset connu).
- **DNT (Do Not Track) header** : respect strict.
- **Logged-in user vs anonymous** : pas de PII ; juste un userId hashé (opt-in).
- **Multi-tab** : événements distincts par tab.
- **Pages privées** : pas de URL contenant PII dans le path.

## Non-goals

- Pas de tracking comportemental fin (heatmap) au MVP (sur-dimensionné).
- Pas d'A/B test au MVP (story v1.x).
- Pas de session replay (privacy).
- Pas de « user identification » avec email (PII).

## Tests à écrire

- **Unit** :

  - `useAnalytics().track()` : payload conforme au schéma
  - DNT respecté : track → no-op si DNT=1

- **Integration** :

  - Mock Plausible endpoint → reception correcte des events

- **E2E** :

  - Parcours signup → vérifier 2 events envoyés
  - Page privée → vérifier URL pas « leakée »

- **Security** :

  - Pas de PII dans payload events (CI grep test)

- **A11y** :

  - Pas d'impact direct.

## Observabilité

- Plausible dashboard est la source de vérité.
- **Métriques** :

  - Compteur `cursus.analytics.events.sent.total`
  - Compteur `cursus.analytics.events.dropped.total` (DNT)

- **Alertes** : aucune (analytics = best-effort).

## Considérations sécurité

- DSAR : Plausible ne stocke pas de PII.
