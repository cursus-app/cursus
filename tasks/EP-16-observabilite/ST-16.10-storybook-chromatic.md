## Contexte business

Le playbook (§3.12) liste Storybook 8 + Chromatic dans les outils premium. Storybook fournit un workshop isolé pour développer les composants (atoms, molecules, organismes) sans contexte app. Chromatic ajoute les tests visuels automatisés : tout pixel-diff inattendu échoue la PR. C'est un pilier de la qualité design system (Premium 1).

## Description fonctionnelle

Storybook 8 installé avec preset Vue 3 + Nuxt. Une story par variant des composants atomes / molécules (Button, Input, Badge, FormField, Card, EmptyState, etc.). Configuration des controls (props), themes (light/dark toggle), backgrounds. Chromatic configé : run sur chaque PR, comparaison visuelle, review UI, accept changes via Chromatic dashboard. Storybook deployé en preview (`stories.cursus.app/<branch>`).

## Critères d'acceptation (Gherkin)

**Scénario 1 — Composant Button avec 4 variants**

```gherkin
Given Button.stories.ts existe
When je lance "pnpm storybook"
Then je vois les 4 variants : primary, secondary, ghost, danger
And les 3 sizes : sm, md, lg
And je peux toggle dark/light
```

**Scénario 2 — Chromatic détecte une régression**

```gherkin
Given une PR change le padding du Button
When Chromatic exécute la PR
Then un visual diff est détecté
And la check PR Chromatic devient orange (review requis)
And le designer peut accepter ou refuser depuis Chromatic UI
```

**Scénario 3 — Snapshot dark + light**

```gherkin
Given une story est marquée parameters.chromatic.modes = ['light', 'dark']
When Chromatic snapshote
Then 2 captures sont prises pour cette story
And diff calculé sur chaque mode
```

## Cas limites à traiter

- **Flaky animation** : disable animations en mode Chromatic.
- **Image externe (CDN)** : mock via decorator.
- **Composant qui dépend du router** : mock router decorator.
- **Composant connecté à Pinia** : story-level provider.
- **PR design system breaking** : Chromatic require approval (block merge).
