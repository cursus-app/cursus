## Contexte business

Soutenance notée selon une grille structurée pour reproductibilité et équité. Sans grille standardisée, deux formateurs notent différemment et le certificat perd sa valeur.

Valeur livrée : grille avec 5 critères par défaut, notation /4 par critère, mention auto-calculée (non validé / validé sous conditions / validé / validé avec mention), workflow vers émission certificat.

## Description fonctionnelle

Formulaire de notation : 5 critères (compréhension technique, qualité livrable, clarté présentation, gestion questions, originalité), notation 0-4 par critère + commentaire global. Score total /20 + mention auto. Re-tentative créée si non validé.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Notation par critère**

```gherkin
Given le formateur entame la soutenance
When il ouvre la grille
Then il voit 5 critères : compréhension technique, qualité livrable, clarté présentation, gestion questions, originalité
```

**Scénario 2 — Calcul mention**

```gherkin
Given il note chaque critère (0-4)
When il soumet
Then le score total /20 est calculé
And la mention est attribuée : <10 non validé, 10-12 validé sous conditions, 13-15 validé, 16-20 validé avec mention
And son commentaire global est sauvegardé
```

**Scénario 3 — Workflow re-tentative**

```gherkin
Given la soutenance est marquée "non validée"
When la grille est soumise
Then une 2ᵉ tentative est créée automatiquement (statut "re-tentative")
```

**Scénario 4 — Workflow certificat**

```gherkin
Given la soutenance est validée
Then le workflow vers émission certificat (EP-10) est déclenché
```

## Cas limites à traiter

- **Note partielle** (pas tous les critères remplis) : refus de submit.
- **Modification post-validation** : interdit (sauf admin avec motif).
- **Grille personnalisée par cursus** : critères propres au cursus si configurés.
- **Co-formateur notant** : audit log indique qui a noté.
