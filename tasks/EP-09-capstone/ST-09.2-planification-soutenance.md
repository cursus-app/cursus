## Contexte business

Soutenance = défense orale du capstone. Au MVP : date manuelle + lien visio externe (Zoom/Meet). Étape critique pour l'émission du certificat.

Valeur livrée : planification 1-clic après soumission finale, invitation par email avec ICS pour ajout au calendrier.

## Description fonctionnelle

Après soumission finale validée, le formateur saisit date prévue + lien visio. ICS généré pour invitation calendrier. Modification de date → notif stagiaire.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Planification**

```gherkin
Given la soumission finale est validée
When le formateur ouvre la fiche capstone
Then il peut saisir : date soutenance, lien visio (Zoom/Meet/etc.)
```

**Scénario 2 — Visualisation stagiaire**

```gherkin
Given la soutenance est planifiée
When le stagiaire consulte
Then il voit la date et le lien
And il reçoit une invitation par email avec ICS
```

**Scénario 3 — Modification de date**

```gherkin
Given la soutenance est planifiée
When le formateur change la date
Then le stagiaire est notifié (in-app + email)
And un nouvel ICS est envoyé
```

## Cas limites à traiter

- **Date dans le passé** : refus validation Zod.
- **Lien visio non-https** : warning.
- **Annulation** : possibilité de unplan.
- **Stagiaire indisponible** : pas géré au MVP (coordination hors-app).
