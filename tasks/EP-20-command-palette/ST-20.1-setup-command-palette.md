## Contexte business

La command palette (Cmd+K) est devenue un standard premium des SaaS modernes (Linear, Notion, Vercel). Elle transforme l'expérience de navigation : un user power peut atteindre n'importe quel cursus, cohorte, stagiaire ou action en 3 touches. C'est un marqueur fort du pilier Premium (01-vision axe Premium-4 "navigation rapide").

Cette story livre la coquille (setup, raccourci, overlay accessible). Le contenu vivant (index + actions) viendra dans ST-20.2 / ST-20.3 / ST-20.5.

## Description fonctionnelle

Installer une lib (par défaut : choisir entre `vue-command-palette` / `cmdk-vue` / le composant `UCommandPalette` de `@nuxt/ui`). Le composant est monté globalement dans `app.vue`. Raccourci global Cmd+K (macOS), Ctrl+K (Windows/Linux). Overlay modal avec input de recherche + liste de résultats sectionnée.

Au MVP de la story : un seul provider "vide" ("Aucun résultat — commence à taper"). L'objectif est d'avoir l'enveloppe focus-trap, animée, accessible, prête à recevoir des providers.

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Cmd+K ouvre la palette
Given un user est sur n'importe quelle page
When il presse Cmd+K (ou Ctrl+K)
Then un overlay modal apparait
And l'input a le focus immédiatement
And un léger fade-in (150ms) anime l'entrée
```

```gherkin
Scénario 2 — Esc ferme
Given la palette est ouverte
When l'user presse Esc
Then la palette se ferme
And le focus retourne sur l'élément d'origine
```

```gherkin
Scénario 3 — Click outside ferme
Given la palette est ouverte
When l'user clique hors de la modal
Then la palette se ferme
And la même logique de focus s'applique
```

## Cas limites à traiter

- **Cmd+K dans un input** : si dans un textarea/input, le raccourci ouvre quand même la palette (intentionnel — c'est un escape rapide).
- **Plusieurs ouvertures rapides** : debounce ou idempotence (l'ouverture ne fait pas mounting multiple).
- **Mobile** : remplacer Cmd+K par un bouton search dans la nav (raccourci clavier inutile).
- **prefers-reduced-motion** : pas d'animation, apparition instantanée.
- **Conflit avec navigateur** : Cmd+K en Firefox = focus address bar — on `preventDefault` notre ouverture.
- **z-index** : la palette doit être au-dessus de tout (modal, toast, banner).

## Tests à écrire

- **A11y axe-core** : focus trap actif, focus return correct, role `dialog` + `aria-modal="true"` + `aria-labelledby`.
- **E2E Playwright** : Cmd+K ouvre, Esc ferme, click outside ferme, retour focus correct.

## Observabilité

- **Logs** : event `cmdk.opened` (info) avec `{ user_id, route, method: 'kbd' | 'click' }`.
- **Métriques** : compteur `cursus.cmdk.opens.total` — cible : > 30 % des power users ouvrent au moins 1 fois par session.

## Considérations sécurité

- Pas d'exécution de code arbitraire (les commandes sont une liste fixe côté client).
- Pas de leak de données PII via index (vient en ST-20.2 avec RLS).

## Considérations accessibilité

- `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointant sur un titre invisible.
- Focus trap rigoureux (Tab cycle dans la modal, Shift+Tab inverse).
- Esc ferme et restore le focus.
- Annonce des résultats via `aria-live="polite"` sur le compteur "N résultats".
- Touche entrée exécute la commande sélectionnée.
- Mobile : alternative cliquable, raccourci clavier optionnel.

## Considérations performance

- Component lazy-loadé (uniquement mounté à la première ouverture).
- Pas de re-render à chaque touche sans debouncing (150 ms debounce sur le filtre).
- Animation CSS GPU-accelerated (transform, opacity).
- Bundle ajouté par la lib : cible < 30 KB gz.
