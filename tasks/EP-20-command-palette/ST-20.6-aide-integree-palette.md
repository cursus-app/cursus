## Contexte business

La command palette gagne en utilité quand elle expose aussi l'aide : un user qui ne sait pas comment fonctionne le harnais peut taper "harnais" et voir une section "Aide" qui propose les articles pertinents ("Comment fonctionne le harnais ?", "Que faire si mon test échoue ?"). Cette story intègre les snippets d'aide au sein même de la palette.

Business value : réduire le ticket de support en exposant les réponses au moment du besoin. Premium-feature d'autosupport.

## Description fonctionnelle

- Source d'aide : fichiers Markdown dans `content/help/**.md` (utilisés aussi par la page `/aide`), indexés en tâche de build.
- Provider dans la palette : section "Aide" avec icône info, max 5 résultats.
- Click ouvre un drawer latéral avec le contenu de l'article (ou redirige vers `/aide/slug`).
- L'index inclut un `title`, `excerpt`, `tags`, `slug`.
- Search : `ILIKE` sur title + excerpt + tags au MVP, FTS via ST-20.4 ensuite.

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Aide sur "harnais"
Given la palette est ouverte
When je tape "harnais"
Then une section "Aide" apparaît avec 3 articles pertinents
And cliquer ouvre un drawer avec le contenu Markdown rendu
```

```gherkin
Scénario 2 — Drawer fermable
Given le drawer d'aide est ouvert
When je presse Esc ou clique X
Then le drawer se ferme
And la palette se ré-ouvre avec ma query (continuité)
```

```gherkin
Scénario 3 — Aide en langue user
Given un user EN tape "validator"
When la palette filtre
Then seuls les articles avec front-matter locale=en remontent
And les articles FR sont absents
```

## Cas limites à traiter

- **Aucun article qui match** : pas de section "Aide" (silencieux, pas de empty state interne).
- **Très long article** : drawer scrollable avec table des matières.
- **Liens internes dans l'article** : ouvrent dans la même app, ferment le drawer.
- **Article qui change** après build : l'index est rebuildé en CI à chaque deploy.
- **Pas d'article FR pour un sujet** : fallback EN avec mention "Available in English".

## Sous-tâches techniques

- **TT-20.6.1** — Convention front-matter Markdown : `title`, `slug`, `tags`, `locale`, `excerpt`.
- **TT-20.6.2** — Script build-time : parser `content/help/**.md` et générer `content/help/index.json`.
- **TT-20.6.3** — Provider "Aide" dans la palette qui filtre l'index par query.
- **TT-20.6.4** — Composant `<HelpDrawer />` qui rend le Markdown (avec `@nuxt/content` ou markdown-it).
- **TT-20.6.5** — Rédiger 10 articles d'aide MVP (qu'est-ce qu'un cursus, comment créer une cohorte, soumettre un livrable, etc.).
- **TT-20.6.6** — Lien vers la page `/aide` (catalogue complet).

## Dépendances

- **Bloqué par** : ST-20.1 (palette), ST-19.1 (i18n pour locale-aware).
- **Bloque** : ST-22.\* (les connecteurs externes ont besoin d'aide rédigée pour configuration).

## Non-goals

- Pas de chatbot AI intégré (story EP-21 si besoin).
- Pas de feedback inline ("Était-ce utile ?") au MVP.
- Pas de versioning des articles d'aide.
- Pas de support vidéo embarquée.

## Tests à écrire

- **Unit Vitest** : parser front-matter valide les champs requis.
- **Integration** : taper "harnais" → résultats "Aide" listent les bons articles.
- **A11y axe-core** : drawer focus trap, Esc ferme.
- **E2E Playwright** : ouvrir palette, chercher aide, lire un article, fermer.
- **Build** : script idempotent, output JSON conform Zod.

## Observabilité

- **Logs** : event `help.article.opened` (info) avec `{ article_slug, source: 'cmdk'|'page', user_role }`.
- **Métriques** : compteur `cursus.help.articles.opened.total` par slug — sert à prioriser quels articles améliorer.
- **Insight** : taux d'abandon d'un article (< 30 s) > 70 % → l'article ne répond pas à la query.

## Considérations sécurité

- Contenu Markdown sanitisé (pas de `<script>`, juste le Markdown standard).
- Liens externes : `rel="noopener noreferrer"`.
- Pas de PII dans les articles (contenu public).

## Considérations accessibilité

- Drawer : `role="dialog"` + `aria-labelledby` titre article.
- Headings du Markdown : `<h2>` (la page parent est `<h1>`).
- Liens internes annoncés comme "étape suivante".
- Contraste typo respecté (token `--color-fg-default`).

## Considérations performance

- Index JSON ≤ 200 KB (gz) — chargé lazy seulement à la 1ère ouverture palette.
- Drawer : mounté à la demande, contenu fetché à la demande.
- Markdown render < 50 ms pour un article moyen (1000 mots).
