## Contexte business

Une command palette qui ne fait que naviguer reste 50 % de sa valeur. Le reste : exécuter des actions ("Créer un cursus", "Inviter un stagiaire", "Exporter CSV"). Cette story branche les actions contextuelles dans la palette — typer "créer" liste les actions de création disponibles selon le contexte et les droits.

Business value : transforme la palette en hub d'action. Un formateur n'a plus besoin de naviguer à la page cursus pour créer ; il presse Cmd+K, tape "créer", choisit.

## Description fonctionnelle

Registre d'actions déclaré côté client : tableau de `Action { id, label, keywords, icon, scope, handler, requiresRole }`. La palette les inclut dans une section "Actions" en haut des résultats. Filtrage selon la query et le contexte (route active, rôle).

Actions MVP : Créer un cursus, Créer une cohorte, Inviter un stagiaire (contextuelle à une cohorte), Exporter CSV (sur cursus / cohortes), Basculer dark/light, Ouvrir l'aide, Se déconnecter, Aller au dashboard, Aller au profil.

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Action contextuelle
Given je suis sur /cohortes/123 en tant que formateur
And j'ouvre la palette
When je tape "inviter"
Then l'action "Inviter un stagiaire dans cette cohorte" apparaît
And Enter ouvre la modal d'invitation pré-rempli avec cohorteId
```

```gherkin
Scénario 2 — Action filtrée par rôle
Given je suis stagiaire
When j'ouvre la palette et tape "créer"
Then aucune action de création de cursus n'est listée
And seules les actions stagiaire pertinentes apparaissent
```

```gherkin
Scénario 3 — Action globale
Given je suis sur n'importe quelle page
When je tape "dark"
Then l'action "Basculer en dark mode" apparaît
And Enter switch le thème immédiatement
```

## Cas limites à traiter

- **Action async** (nécessite confirmation) : la palette ne ferme pas avant succès, affiche un état loading.
- **Action qui échoue** : toast d'erreur, palette reste ouverte avec focus restoré.
- **Action redondante avec une commande clavier** (Cmd+B = dark mode) : afficher le raccourci à côté de l'action.
- **Action nécessitant un contexte manquant** : ne pas afficher (ex: "Inviter dans cette cohorte" si pas sur une page cohorte).
- **Action destructive** (Se déconnecter) : confirmation modale après sélection ou mécanisme "hold to confirm".
- **Beaucoup d'actions match** : trier par récence d'usage + popularité globale.

## Sous-tâches techniques

- **TT-20.3.1** — Registry `actions/index.ts` : type `Action`, helper `defineAction`.
- **TT-20.3.2** — Provider dans la palette : merge actions + résultats de recherche.
- **TT-20.3.3** — Filtrage selon `requiresRole` (utilise `useAuth().user.role`).
- **TT-20.3.4** — Filtrage selon `scope` (ex: route param `cohorteId` disponible).
- **TT-20.3.5** — Définir 10 actions MVP avec handlers (créer cursus ouvre modal, etc.).
- **TT-20.3.6** — Affichage raccourci clavier à droite de l'action si défini.
- **TT-20.3.7** — Persistance MRU (Most Recently Used) : top 5 actions récents en tête quand query vide.
- **TT-20.3.8** — Tests des handlers (ouverture modal, navigation, switch thème).

## Dépendances

- **Bloqué par** : ST-20.1 (palette setup), ST-20.2 (index searchable), ST-02.4 (RBAC).
- **Bloque** : aucune — amélioration UX.

## Non-goals

- Pas d'actions personnalisées par user (custom shortcuts).
- Pas de chain d'actions ("do A then B").
- Pas d'historique persistant cross-device (juste localStorage).
- Pas de génération d'actions par IA (future EP-21).

## Tests à écrire

- **Unit Vitest** : `defineAction` valide les champs ; filter applique RBAC et scope.
- **Integration** : action "Créer cursus" ouvre bien la modal builder.
- **A11y axe-core** : actions navigables au clavier, role `option` correctement annoncé.
- **E2E Playwright** : parcours formateur : Cmd+K → taper "créer" → Enter → modal apparait.
- **RBAC** : test stagiaire ne voit pas l'action admin.

## Observabilité

- **Logs** : event `cmdk.action_executed` (info) avec `{ action_id, scope, user_role, success }`.
- **Métriques** : compteur `cursus.cmdk.actions.executed.total` labellisé par `action_id` — sert à prioriser les actions les plus utilisées.
- **Insight** : si une action a > 50 executions / semaine, candidate à un raccourci clavier dédié.

## Considérations sécurité

- Chaque handler vérifie le rôle côté serveur en plus de la palette (defense in depth).
- Actions destructives : confirmation requise (modale ou hold).
- Pas d'exécution de code arbitraire (handlers sont des fonctions enregistrées).

## Considérations accessibilité

- Chaque action a un `aria-label` complet et un raccourci visible.
- Annonce de l'exécution ("Création du cursus en cours…") via `aria-live`.
- Actions destructives ont un libellé explicite ("Supprimer définitivement").

## Considérations performance

- Filter en mémoire (< 50 actions, instantané).
- Handler async ne bloque pas l'UI (loading spinner inline).
- Persistance MRU debouncée (write localStorage max toutes les 2s).
