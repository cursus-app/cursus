## Contexte business

La palette (Cmd+K) couvre 80 % des besoins de raccourcis, mais quelques raccourcis directs accelèrent les actions vraiment fréquentes : `/` pour focus la recherche dans la palette, `g h` (go home) pour aller au dashboard, `g c` pour cohortes, `?` pour ouvrir l'aide, `Cmd+/` pour switch dark/light. Cette story livre l'infrastructure de shortcuts globaux + une dizaine de bindings de base.

Business value : un power user navigue 2-3× plus vite, perception de produit haute densité. Marqueur Premium.

## Description fonctionnelle

- Composable `useKeyboardShortcuts(config)` qui prend `{ key, modifiers, handler, description, scope }` et enregistre dans un registry global.
- Service global de capture `window.addEventListener('keydown')` qui dispatch selon la combo.
- Page d'aide / cheat sheet (`/keyboard-shortcuts` ou modal accessible via `?`) qui liste tous les raccourcis enregistrés avec leur description.
- Detection de la plateforme (macOS vs autre) pour afficher Cmd vs Ctrl.
- Bindings MVP : Cmd+K (palette, déjà défini), `?` (cheat sheet), `g h` (home), `g c` (cohortes), `g p` (portfolio), `Cmd+/` (theme), `Esc` (close any modal), `n c` (nouvelle cohorte), `n s` (nouveau stagiaire).

## Critères d'acceptation (Gherkin)

```gherkin
Scénario 1 — Raccourci go home
Given un user est sur /portfolio
When il presse g puis h dans 500 ms
Then il est redirigé vers /dashboard
And un toast discret confirme "Go home"
```

```gherkin
Scénario 2 — Cheat sheet via ?
Given le user presse ?
When aucun input n'a le focus
Then une modal liste tous les raccourcis disponibles
And les raccourcis sont groupés par scope (Global, Cursus, Cohortes)
And les raccourcis non-applicables au rôle sont masqués
```

```gherkin
Scénario 3 — Pas de capture dans un input
Given le focus est dans un <textarea>
When je tape "g h"
Then aucun raccourci ne se déclenche
And le texte "g h" est tapé normalement
```

## Cas limites à traiter

- **Combinaison de touches en séquence (**`g h`) : timeout 800 ms entre les deux, sinon abandon.
- **Conflict avec navigateur** : ne pas surcharger Cmd+T, Cmd+W, Cmd+R (browser reserved).
- **Modifier keys** : gérer Cmd (macOS), Ctrl (Windows/Linux), MetaKey detection.
- **Plusieurs handlers pour le même raccourci** : dernière inscription gagne, warn en console dev.
- **Raccourcis selon rôle** : ne pas afficher dans la cheat sheet ce que le user ne peut pas faire.
- **Internationalisation** : clés clavier identiques mais descriptions traduites.

## Non-goals

- Pas de raccourcis personnalisables par user au MVP.
- Pas de chord 3-keystrokes (`g g g`).
- Pas de support manettes / accessibilité switch-control au MVP (a11y stories dédiées si besoin).
- Pas de capture en mode plein écran d'une vidéo.

## Tests à écrire

- **Unit Vitest** : registry inscription / désinscription, formatKeystroke macOS vs Windows.
- **Integration** : taper `g h` redirige bien, taper dans un input ne déclenche pas.
- **A11y axe-core** : cheat sheet a focus trap correct, `<kbd>` semantique pour les touches.
- **E2E Playwright** : parcours utilisateur avec 3 raccourcis successifs.
- **Security** : pas d'exécution d'event arbitraire (handlers déjà enregistrés).

## Observabilité

- **Logs** : event `kbd.shortcut.triggered` (info) avec `{ shortcut, scope, user_role }`.
- **Métriques** : compteur `cursus.kbd.shortcut.triggered.total` labellisé par `shortcut`.
- **Insight** : si un raccourci ne sert jamais sur 30 jours, candidat à supprimer ; si très utilisé sans raccourci, candidat à ajouter.

## Considérations sécurité

- Pas de surface sécurité (handlers locaux et registerés).
- Les raccourcis admin sont gatés par rôle côté serveur (defense in depth).

## Considérations accessibilité

- Cheat sheet utilise `<kbd>` pour chaque touche (semantique).
- Navigation entièrement clavier dans la cheat sheet.
- Pas de raccourci qui interfère avec lecteurs d'écran (NVDA `Insert+`, JAWS, VoiceOver `Ctrl+Option`).
- Respect `prefers-reduced-motion` pour les toast d'action.
- Annonce `aria-live="polite"` du toast confirmant l'action.

## Considérations performance

- Listener global : 1 event handler, dispatch O(1) via Map.
- Cheat sheet : lazy mount (uniquement à la première ouverture).
- Pas de boucle ou polling.
