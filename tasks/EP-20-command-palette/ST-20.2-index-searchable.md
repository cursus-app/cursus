## Contexte business

La command palette (ST-20.1) vide est inutile. Cette story remplit son index avec les entités principales : cursus, cohortes, stagiaires, modules, alertes. L'index est ce qui transforme la palette d'une "barre vide" en outil de productivité ("trouver Mehdi en 3 touches").

Business value : un formateur gérant 5 cohortes économise plusieurs minutes / jour en navigation. C'est un Premium-feature visible immédiatement.

## Description fonctionnelle

Provider côté client + endpoint serveur `GET /api/search?q=...&scope=...&limit=10` qui agrège des résultats par type. La palette affiche les résultats par section (Cursus / Cohortes / Stagiaires / Modules / Alertes), chacune navigable au clavier (via `UCommandPalette` de @nuxt/ui v4).

Côté client : composable `useSearch(query)` debouncé 250 ms, cache LRU par query, abort des fetches obsolètes.

Côté serveur : requête Prisma utilisant `contains` (case-insensitive) pour le MVP, basculer sur Postgres FTS dans ST-20.4 (améliore relevance + perf). RLS strict : un formateur ne voit que ses cohortes / stagiaires, un admin voit tout.

## Critères d'acceptation (Gherkin)


## Cas limites à traiter

- **Query vide** : afficher "Récents" (5 derniers éléments ouverts par l'user) plutôt qu'un vide.
- **Query 1 caractère** : ne pas requêter (trop de bruit), attendre 2 caractères.
- **Query qui change rapidement** : abort des fetches obsolètes via `AbortController`.
- **Résultats nuls** : empty state "Aucun résultat pour 'xyz'" avec suggestion ("Essaie un autre mot-clé").
- **Beaucoup de résultats** : limite 10 par section, lien "voir tout" qui ouvre la page de recherche complète (ST-20.4).
- **Accents et casse** : recherche insensible (`mode: 'insensitive'` Prisma ou extension Postgres `unaccent`).

## Sous-tâches techniques

- **TT-20.2.1** — Endpoint `GET /api/search` avec Zod validation (`q: string min 2`, `scope?: enum`, `limit: number max 20`).
- **TT-20.2.2** — Requêtes Prisma multi-models (Cursus, Cohorte, User, Module, Alert) avec `contains` + RLS appliquée via rôle `cursus_app` + middleware applicatif.
- **TT-20.2.3** — Index Postgres : `idx_cursus_title_trgm` (extension pg_trgm) pour accélérer les recherches — créés via migration Prisma `--create-only` puis SQL brut.
- **TT-20.2.4** — Composable `useSearch(query)` avec debounce 250 ms (`@vueuse/nuxt`), cache LRU 20 entries, AbortController.
- **TT-20.2.5** — Provider dans la palette `UCommandPalette` : map les résultats vers les sections, icônes Tabler (`tabler:school`, `tabler:users`, `tabler:user`, `tabler:books`, `tabler:bell`).
- **TT-20.2.6** — Récents : helper `useRecentItems()` (localStorage, top 5 par scope).
- **TT-20.2.7** — Tests RLS négatifs.

## Non-goals

- Pas de filtre avancé (date, tag) au MVP — vient en ST-20.4.
- Pas de recherche cross-organisations (single-tenant).
- Pas de boost par récence ou popularité — ranking simple alphabétique au MVP, amélioré en ST-20.4.
- Pas de recherche dans le contenu des PDFs.

## Tests à écrire

- **Unit Vitest 4.1** : `useSearch` debounce correct, abort fonctionne.
- **Integration** : `GET /api/search?q=Mehdi` retourne résultats RLS-filtrés pour un formateur.
- **RLS négatif** : formateur A ne voit pas un stagiaire du formateur B.
- **E2E Playwright** : ouvrir palette, taper "Mehdi", cliquer → redirige bien.
- **Performance** : query p95 < 200 ms sur 10k records.
- **Security** : injection (caractères spéciaux SQL) bloquée par Prisma paramétré.

## Observabilité

- **Logs** : event `search.query` (info, sampling 10 %) avec `{ q_length, scope, results_count, latency_ms }` (pas la query brute pour PII).
- **Métriques** : histogram `cursus.search.latency_ms`, compteur `cursus.search.no_results.total`.
- **Alerte** : p95 latency > 500 ms → enquête.
- **Insight** : taux de no-results > 30 % → query intent mal compris, ajuster index.

## Considérations sécurité

- RLS Supabase active sur toutes les tables indexées (effective via rôle `cursus_app` que Prisma utilise).
- Rate limit : 60 queries / minute / user (anti-scraping).
- Param Zod strict, pas d'injection SQL (Prisma paramétré).
- Log de la query : hash si > 10 chars, pas la query brute (PII potentielle).

## Considérations accessibilité

- `UCommandPalette` (@nuxt/ui v4) gère déjà ARIA combobox + announces.
- Résultats annoncés via `aria-live="polite"` ("5 résultats trouvés").
- Sections avec `<h3>` ou `aria-labelledby`.
- Indicateur de loading visuel + textuel (USkeleton + "Recherche en cours…").
- Navigation clavier 100 % couverte (↑↓, Enter, Esc).

## Considérations performance

- Index `pg_trgm` GIN sur les colonnes `title`, `displayName`.
- Cache LRU côté client (évite refetch pour query déjà vue).
- Limite serveur 10 par section, 50 total max.
- Cible : p95 latency ≤ 200 ms pour 10k records.
- Payload réponse < 30 KB (id + label + type + URL minimum).
