## Contexte business

Le portfolio est la **North Star** du produit : prouve que le stagiaire sait reproduire. URL publique pour partager avec recruteurs/réseau. Sans portfolio, le certificat seul ne suffit pas.

Valeur livrée : URL publique `/p/<slug>` avec tout le parcours visible (livrables avec repos, badges, capstone, lien certificat), privé tant que capstone non validé.

## Description fonctionnelle

URL publique `/p/<slug>` accessible sans login. SSR + SEO complet (OG, JSON-LD). Privé tant que capstone non validé. Toggle privé/public par le stagiaire. Stats de visite (privates).

## Critères d'acceptation (Gherkin)

**Scénario 1 — Portfolio public après capstone**

```gherkin
Given un stagiaire a validé son capstone
When son portfolio est généré
Then une URL publique /p/<prenom>-<initiale> est créée
And la page affiche : photo, nom, cursus, dates, liste livrables avec liens repos, badges, capstone, lien certificat
```

**Scénario 2 — Portfolio non publié**

```gherkin
Given un stagiaire n'a pas validé son capstone
When quelqu'un visite son URL
Then la page renvoie "Portfolio non encore publié"
```

**Scénario 3 — Toggle privé**

```gherkin
Given le stagiaire bascule "privé"
Then l'URL renvoie 404
But le certificat reste vérifiable (ST-10.3)
```

## Cas limites à traiter

- **Slug en conflit** : suffix numérique (`karim-d-2`).
- **Stagiaire supprime son compte** : portfolio anonymisé (initiales) mais certificat reste vérifiable.
- **Stagiaire modifie son nom** : slug ne change pas (stable).
- **Capstone révoqué** : portfolio bascule en privé auto.

## Considérations sécurité

- RLS : stagiaire toggle privacy.
- Public page = pas d'auth (cache CDN agressif).
- Pas de PII non-consenties (juste prénom + initiale par défaut).
- Rate limit sur scrapers (Cloudflare).

## Considérations accessibilité

- **WCAG AAA** : c'est la vitrine, on monte le niveau.

- Liens repos : `target="_blank" rel="noopener"`.
- Structure semantique forte (heading hierarchy).

## Considérations performance

- **LCP < 1.5s** (SSR + cache CDN).
- Images optimisées (Supabase Image Transform).
- Lazy load des sections sous le fold.
