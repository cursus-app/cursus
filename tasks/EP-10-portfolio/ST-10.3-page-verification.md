## Contexte business

Le recruteur doit pouvoir vérifier en **5 secondes** l'authenticité d'un certificat. Cible **A11y AAA**. Sans cette page, le certificat signé est inutile (signature non vérifiable par un tiers).

Valeur livrée : page publique de vérification ultra-rapide, accessible sans login, design soigné (vitrine externe), validation cryptographique côté serveur.

## Description fonctionnelle

Page publique `/verify/:token` accessible sans login. Validation cryptographique côté serveur. Affiche : Valide / Révoqué / Invalide. Identité, cursus, date émission. Rate limit par IP anti-scraping.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Certificat valide**

```gherkin
Given un recruteur scanne le QR ou clique sur un lien
When il arrive sur /verify/<token>
Then la page valide la signature
And affiche le statut : Valide
And affiche : nom du stagiaire, cursus, date émission, mention
```

**Scénario 2 — Certificat révoqué**

```gherkin
Given le certificat est révoqué
When le token est validé
Then statut = "Révoqué" avec message neutre (pas de motif détaillé)
```

**Scénario 3 — Token forgé**

```gherkin
Given le token est forgé / invalide
When la validation échoue
Then statut = "Invalide" sans révéler pourquoi
And pas d'info qui aiderait un attaquant
```

## Cas limites à traiter

- **Token modifié 1 caractère** : signature invalide → "Invalide".
- **Volumétrie** : rate limit par IP (anti-scraping).
- **Stagiaire qui supprime son compte** : identité anonymisée mais certificat vérifiable.
- **Accès sans QR (lien direct)** : OK.

## Observabilité

- **Métriques** : `verifications.requested.total`, taux Valide/Révoqué/Invalide.
- **Logs** : `verification.checked` (sans token complet).
- **Alertes** : pic anormal de verifications = potential attack.

## Considérations sécurité

- Rate limit strict (10/min/IP).
- Pas d'énumération possible.
- Messages génériques sur "Invalide" (no info leak).
- Constant-time signature check (no timing).
- CSRF non applicable (GET only).

## Considérations accessibilité

- **WCAG AAA** : c'est l'écran le plus critique.
- Statut : icône + texte + couleur.
- Annonce du résultat via `aria-live`.
- Navigation clavier complète.

## Considérations performance

- **LCP < 1s** (page minimaliste, SSR, cache CDN).
- Vérification serveur : < 100ms.
- Pas de JS lourd (vitrine simple).

## Definition of Done spécifique

- E2E : scan QR → vérif < 5s end-to-end.
- Lighthouse A11y AAA + Performance > 95.
- Test tokens forgés : rejet.
