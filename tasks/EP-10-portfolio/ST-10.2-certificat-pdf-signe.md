## Contexte business

Le certificat est le **livrable légal** du cursus. Doit être professionnel, **signé numériquement**, vérifiable par un tiers (recruteur). Sans signature, le certificat est falsifiable.

Valeur livrée : PDF généré automatiquement à la validation du capstone, signé cryptographiquement (Ed25519), QR code vers page de vérification publique, stocké dans Supabase Storage.

## Description fonctionnelle

À la validation du capstone, génération automatique d'un PDF via Puppeteer en Nuxt 4 server route. Template configurable via Vue Email (`@vue-email/render` ^0.0.10). Signature cryptographique Ed25519. QR vers vérification publique. Email avec PDF attaché.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Génération automatique**

```gherkin
Given un capstone est validé
When la cérémonie de notation est soumise
Then un PDF certificat est généré (Puppeteer + Vue Email)
And le PDF contient : identité, cursus, dates, modules validés, mention, lien portfolio, QR code
And le PDF est signé numériquement (clé asymétrique Ed25519)
And il est stocké dans Supabase Storage
And le stagiaire reçoit un email avec le PDF
```

**Scénario 2 — Vérification publique via QR**

```gherkin
Given un recruteur scanne le QR
When il atterrit sur /verify/<token>
Then la page publique affiche : Valide / Révoqué, identité (consentement), cursus, date émission
And la signature est vérifiée côté serveur
```

**Scénario 3 — Forgerie détectée**

```gherkin
Given un PDF modifié manuellement
When sa signature est vérifiée
Then elle est invalide → statut "Invalide"
```

## Cas limites à traiter

- **Certificat à régénérer** (correction de typo) : nouvelle version, ancien révoqué.
- **Stagiaire supprime son compte** : certificat reste vérifiable mais identité anonymisée.
- **Clé privée compromise** : rotation clé, tous les certificats marqués "à re-signer".
- **Capstone révoqué** : certificat marqué révoqué.

## Considérations sécurité

- **Clé privée** stockée chiffrée dans Vault Supabase.
- **Ed25519** : robuste, rapide, standard.
- **Hash SHA256** du contenu PDF + signature.
- **Audit log** sur chaque émission.
- **Rotation clé** : stratégie documentée (key versioning).

## Considérations accessibilité

- PDF : tagé (accessible PDF/UA).
- Alt text sur QR code.
- Email avec PDF : sujet clair.

## Considérations performance

- Génération : < 5s (Puppeteer cold start).
- Stockage : compressé (PDF/A optimisé).

## Definition of Done spécifique

- E2E capstone validé → PDF reçu par email.
- Test forgerie : modif PDF → signature invalide.
- QR code testable end-to-end.
