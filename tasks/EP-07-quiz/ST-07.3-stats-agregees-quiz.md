## Contexte business

Aider le formateur à détecter les notions mal comprises au niveau de la cohorte (pas du stagiaire individuel). Anti-stigmatisation : pas d'affichage nominatif sauf écart > 2 sigma.

Valeur livrée : insight rapide sur les questions mal comprises pour ajuster la pédagogie.

## Description fonctionnelle

Vue agrégée par quiz : score moyen, taux de réussite par question. Anonyme par défaut. Pas de classement individuel.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Affichage des stats**

```gherkin
Given un formateur consulte les stats d'un quiz
When il ouvre la vue stats
Then il voit le score moyen de la cohorte
And le taux de réussite question par question
And pas d'affichage nominatif sauf écart > 2 sigma
```

**Scénario 2 — Quiz sans tentative**

```gherkin
Given un quiz sans tentative
When le formateur consulte
Then un message "Pas encore de données" s'affiche
```

**Scénario 3 — Question problématique**

```gherkin
Given une question a < 30% de taux de réussite
Then elle est highlightée en rouge
And un CTA "Réviser cette question" est offert
```

## Cas limites à traiter

- **Quiz sans tentative** : message "Pas encore de données".
- **< 5 répondants** : message "Stats peu représentatives (anonymat)."
- **Question avec 100% réussite** : marquée "trop facile".
