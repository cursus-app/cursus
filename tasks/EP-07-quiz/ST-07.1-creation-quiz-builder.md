## Contexte business

Le formateur crée un quiz attaché à un module : 5-10 questions, QCM (1+ bonnes réponses) ou texte court. Quiz = check-in léger de compréhension, pas un examen (anti-pilier respecté : pas d'anti-triche, pas de timer dur).

Valeur livrée : assemblage rapide d'un quiz avec preview, randomisation au passage (10 questions tirées aléatoirement si > 10), explication post-réponse pour apprentissage.

## Description fonctionnelle

Sous-section du `ModuleEditor`. Liste de questions ajoutables. Chaque question : type (QCM, texte court), énoncé, options (si QCM), bonne(s) réponse(s), explication post-réponse. Validation : pas de bonne réponse cochée = erreur.

## Critères d'acceptation (Gherkin)

**Scénario 1 — Création de questions**

```gherkin
Given un formateur édite un module
When il ajoute un quiz et 7 questions QCM
Then chaque question a énoncé, options, bonnes réponses cochées, explication
```

**Scénario 2 — Question texte court**

```gherkin
Given une question texte court
When il définit la réponse attendue
Then il peut configurer la tolérance (exact match insensible casse, ou validation formateur post)
```

**Scénario 3 — Randomisation au passage**

```gherkin
Given le quiz a >10 questions
When le stagiaire le passe
Then 10 questions sont tirées aléatoirement
```

## Cas limites à traiter

- **Quiz sans question** : ne peut pas être assigné.
- **Question sans bonne réponse cochée** : erreur de validation.
- **Question avec toutes les réponses correctes** : autorisé.
- **Modification d'un quiz déjà passé** : warning fort, versioning impact.

## Non-goals (hors scope)

- Pas de question avec image au MVP (uniquement texte).
- Pas de mathématiques / formules au MVP.
- Pas de questions à réponse longue (essay).
- Pas de banque de questions partagée.

## Tests à écrire

- **Unit** : validation question (au moins 1 bonne réponse).
- **E2E (Playwright)** : créer quiz + le voir en mode stagiaire.
- **Unit** : tirage aléatoire si > 10 questions (reproductible avec seed pour test).

## Observabilité

- **Métriques** : nombre moyen de questions par quiz (cible 5-10).

## Considérations sécurité
